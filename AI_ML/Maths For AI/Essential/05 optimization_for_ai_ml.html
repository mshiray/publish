Gradient Descent Variants

ML LLM Training Optimization

Theory

Gradient Descent is the backbone of nearly all neural-network training. It minimizes a loss function L(θ) by iteratively moving parameters θ in the direction opposite to the gradient ∇L. The intuition: stand on a hilly landscape, always step downhill. Three main flavors exist — Batch GD (whole dataset per step), Stochastic GD (one sample per step), and Mini-Batch GD (small batches). Modern deep learning almost exclusively uses Mini-Batch SGD because it provides noisy-but-fast gradient estimates that also act as implicit regularization.

Core Update Rule
θ_(t+1) = θ_t − η · ∇_θ L(θ_t ; x_batch, y_batch)

where η = learning rate, ∇_θ L = gradient of loss w.r.t. parameters

Real-World Usage

  • GPT / LLaMA pre-training: Mini-batch SGD with batch sizes 256–4096 tokens over trillions of tokens
  • Image classifiers (ResNet, ViT): Mini-batch GD on ImageNet with gradient accumulation when GPU VRAM is limited
  • Recommendation systems (Netflix, YouTube): Stochastic GD to update embedding tables online as new interaction data arrives
  • Financial risk models: Batch GD for stable parameter estimates on fixed quarterly datasets
gradient_descent_variants.py Python 
import numpy as np
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset

# ── Pure NumPy: Batch vs Mini-Batch vs SGD ──────────────────────────

class LinearModel:
    def __init__(self, n_features):
        self.W = np.random.randn(n_features, 1) * 0.01
        self.b = np.zeros((1,))

    def predict(self, X):
        return X @ self.W + self.b

    def mse_loss(self, X, y):
        preds = self.predict(X)
        return np.mean((preds - y) ** 2)

    def gradients(self, X, y):
        n = len(y)
        preds = self.predict(X)
        err   = preds - y
        dW    = (2 / n) * X.T @ err   # ∂L/∂W
        db    = (2 / n) * np.sum(err)  # ∂L/∂b
        return dW, db


def batch_gd(model, X, y, lr=0.01, epochs=100):
    """Batch GD — uses entire dataset each step (stable, slow for big data)"""
    losses = []
    for _ in range(epochs):
        dW, db = model.gradients(X, y)
        model.W -= lr * dW
        model.b -= lr * db
        losses.append(model.mse_loss(X, y))
    return losses


def sgd(model, X, y, lr=0.01, epochs=100):
    """SGD — one sample per step (noisy, fast updates)"""
    n, losses = len(y), []
    for _ in range(epochs):
        idx  = np.random.permutation(n)
        X, y = X[idx], y[idx]
        for i in range(n):
            Xi = X[i:i+1]
            yi = y[i:i+1]
            dW, db = model.gradients(Xi, yi)
            model.W -= lr * dW
            model.b -= lr * db
        losses.append(model.mse_loss(X, y))
    return losses


def mini_batch_gd(model, X, y, lr=0.01, batch_size=32, epochs=100):
    """Mini-Batch GD — best of both worlds (used in all modern DL)"""
    n, losses = len(y), []
    for _ in range(epochs):
        idx  = np.random.permutation(n)
        X, y = X[idx], y[idx]
        for i in range(0, n, batch_size):
            Xb = X[i : i + batch_size]
            yb = y[i : i + batch_size]
            dW, db = model.gradients(Xb, yb)
            model.W -= lr * dW
            model.b -= lr * db
        losses.append(model.mse_loss(X, y))
    return losses


# ── PyTorch: Mini-Batch with DataLoader (production pattern) ─────────
def train_pytorch_minibatch(X_np, y_np, lr=1e-3, batch_size=64, epochs=20):
    X_t = torch.tensor(X_np, dtype=torch.float32)
    y_t = torch.tensor(y_np, dtype=torch.float32).unsqueeze(1)

    dataset    = TensorDataset(X_t, y_t)
    loader     = DataLoader(dataset, batch_size=batch_size, shuffle=True)
    model      = nn.Linear(X_np.shape[1], 1)
    criterion  = nn.MSELoss()
    optimizer  = torch.optim.SGD(model.parameters(), lr=lr)

    for epoch in range(epochs):
        epoch_loss = 0
        for Xb, yb in loader:
            optimizer.zero_grad()      # clear accumulated gradients
            preds = model(Xb)
            loss  = criterion(preds, yb)
            loss.backward()            # compute ∂L/∂θ via autograd
            optimizer.step()           # θ = θ − η·∇L
            epoch_loss += loss.item()
        print(f"Epoch {epoch+1}: loss={epoch_loss/len(loader):.4f}")


# ── Gradient accumulation for large effective batch size ─────────────
def train_with_grad_accumulation(model, loader, optimizer, accum_steps=4):
    """Simulates batch_size * accum_steps without extra VRAM — critical for LLMs"""
    optimizer.zero_grad()
    for step, (Xb, yb) in enumerate(loader):
        loss = nn.MSELoss()(model(Xb), yb) / accum_steps  # scale loss
        loss.backward()
        if (step + 1) % accum_steps == 0:
            optimizer.step()
            optimizer.zero_grad()

Adam & AdamW Optimizer

LLM Fine-tuning Deep Learning Optimization

Theory

Adam (Adaptive Moment Estimation) combines momentum with per-parameter adaptive learning rates. It tracks two exponential moving averages of the gradient: the first moment m (mean) and the second moment v (uncentered variance). By dividing the step by √v, parameters with large or noisy gradients take smaller steps. Bias correction (dividing by 1−β^t) prevents underestimation early in training. AdamW decouples weight decay from the gradient update — a critical fix because standard Adam's L2 regularization is scale-dependent and doesn't work correctly.

Adam Update Equations
m_t = β₁·m_(t-1) + (1−β₁)·g_t ← 1st moment (momentum)
v_t = β₂·v_(t-1) + (1−β₂)·g_t² ← 2nd moment (adaptive scale)
m̂_t = m_t / (1−β₁ᵗ) ← bias correction
v̂_t = v_t / (1−β₂ᵗ)
θ_(t+1) = θ_t − η · m̂_t / (√v̂_t + ε) ← parameter update

AdamW adds: θ_(t+1) -= η · λ · θ_t ← decoupled weight decay

Defaults: β₁=0.9, β₂=0.999, ε=1e-8, λ=0.01

Real-World Usage

  • GPT-4, LLaMA, Mistral: AdamW with β₁=0.9, β₂=0.95 (lower β₂ for stability at large scale)
  • BERT fine-tuning: AdamW + linear warmup + weight decay 0.01 on all non-bias params
  • Stable Diffusion: Adam for U-Net training; 8-bit Adam (bitsandbytes) to halve optimizer state memory
  • Fraud detection models: Adam's fast convergence lets data scientists iterate quickly on tabular neural nets
adam_adamw.py Python 
import numpy as np
import torch
import torch.nn as nn
from transformers import AdamW, get_linear_schedule_with_warmup

# ── NumPy Adam from scratch ───────────────────────────────────────────
class AdamOptimizer:
    def __init__(self, params, lr=1e-3, betas=(0.9, 0.999), eps=1e-8):
        self.params = params
        self.lr     = lr
        self.b1, self.b2 = betas
        self.eps    = eps
        self.t      = 0
        self.m      = [np.zeros_like(p) for p in params]  # 1st moment
        self.v      = [np.zeros_like(p) for p in params]  # 2nd moment

    def step(self, grads):
        self.t += 1
        for i, (p, g) in enumerate(zip(self.params, grads)):
            self.m[i] = self.b1 * self.m[i] + (1 - self.b1) * g
            self.v[i] = self.b2 * self.v[i] + (1 - self.b2) * g**2
            m_hat = self.m[i] / (1 - self.b1 ** self.t)  # bias-corrected
            v_hat = self.v[i] / (1 - self.b2 ** self.t)
            p     -= self.lr * m_hat / (np.sqrt(v_hat) + self.eps)


# ── PyTorch AdamW — standard LLM fine-tuning setup ───────────────────
def setup_adamw_for_llm(model, lr=2e-5, weight_decay=0.01):
    # Don't apply weight decay to biases and LayerNorm parameters
    no_decay = ["bias", "LayerNorm.weight", "layer_norm.weight"]
    param_groups = [
        {
            "params": [p for n, p in model.named_parameters()
                       if not any(nd in n for nd in no_decay)],
            "weight_decay": weight_decay,
        },
        {
            "params": [p for n, p in model.named_parameters()
                       if any(nd in n for nd in no_decay)],
            "weight_decay": 0.0,
        },
    ]
    optimizer = torch.optim.AdamW(param_groups, lr=lr)
    return optimizer


# ── 8-bit Adam: halves memory for large model training ────────────────
try:
    import bitsandbytes as bnb
    def get_8bit_adam(model, lr=1e-4):
        """Quantizes optimizer states to 8-bit — critical for 7B+ models"""
        return bnb.optim.Adam8bit(model.parameters(), lr=lr,
                                   betas=(0.9, 0.995))
except ImportError:
    print("Install bitsandbytes for 8-bit Adam: pip install bitsandbytes")


# ── Full training loop with AdamW + linear warmup ─────────────────────
def finetune_loop(model, train_loader, num_epochs=3, warmup_ratio=0.06):
    optimizer      = setup_adamw_for_llm(model)
    total_steps    = len(train_loader) * num_epochs
    warmup_steps   = int(total_steps * warmup_ratio)
    scheduler      = get_linear_schedule_with_warmup(
        optimizer, warmup_steps, total_steps
    )
    criterion      = nn.CrossEntropyLoss()

    model.train()
    for epoch in range(num_epochs):
        for batch in train_loader:
            input_ids, labels = batch["input_ids"], batch["labels"]
            optimizer.zero_grad()
            outputs = model(input_ids, labels=labels)
            loss    = outputs.loss
            loss.backward()
            torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)  # clip
            optimizer.step()
            scheduler.step()
        print(f"Epoch {epoch+1} done")

Learning Rate Schedules

LLM Training Optimization

Theory

A fixed learning rate rarely works optimally throughout training. Too large → diverges or oscillates near minima. Too small → painfully slow. Schedules dynamically adjust η during training. Warmup ramps η from near-zero to target over the first steps, preventing early instability when gradients are noisy. Cosine annealing smoothly decays η following a cosine curve, letting the model settle into sharp minima. Cyclical LR periodically resets η, helping escape local minima. Cosine with restarts (SGDR) is the modern gold standard for LLMs.

Cosine Annealing Formula
η_t = η_min + ½·(η_max − η_min)·(1 + cos(π·t / T_max))

Warmup: η_t = η_target · (t / t_warmup) for t < t_warmup
Decay: η_t = η_target · (1 − t/T) linear decay

Real-World Usage

  • GPT-3 / LLaMA pre-training: Cosine decay from peak lr 3e-4 to 10% over trillions of tokens
  • ViT (Vision Transformer): 10k-step warmup followed by cosine annealing on ImageNet-21K
  • Cyclical LR in tabular ML: FastAI's 1-cycle policy for tabular neural nets in Kaggle competitions
  • RLHF (ChatGPT reward models): Linear warmup + linear decay to stabilize PPO training
lr_schedules.py Python 
import math
import torch
from torch.optim.lr_scheduler import (
    CosineAnnealingLR, CosineAnnealingWarmRestarts,
    OneCycleLR, ReduceLROnPlateau
)

# ── Manual cosine schedule with warmup ───────────────────────────────
def cosine_lr_with_warmup(step, warmup_steps, total_steps,
                           lr_max=3e-4, lr_min=3e-5):
    if step < warmup_steps:
        return lr_max * step / warmup_steps          # linear ramp
    progress = (step - warmup_steps) / (total_steps - warmup_steps)
    return lr_min + 0.5 * (lr_max - lr_min) * (1 + math.cos(math.pi * progress))

# visualise schedule
total = 10_000
lrs   = [cosine_lr_with_warmup(t, 500, total) for t in range(total)]


# ── PyTorch built-in schedulers ───────────────────────────────────────
def build_schedulers(optimizer, train_loader, epochs=10):
    steps_per_epoch = len(train_loader)

    # 1. Cosine Annealing — smooth decay, no restarts
    cosine = CosineAnnealingLR(optimizer, T_max=epochs * steps_per_epoch,
                                eta_min=1e-6)

    # 2. Cosine with warm restarts (SGDR) — periodically resets LR
    sgdr   = CosineAnnealingWarmRestarts(optimizer, T_0=steps_per_epoch,
                                          T_mult=2)  # doubles period each restart

    # 3. One-Cycle LR (Smith 2019) — fast.ai standard, great for tabular
    one_cycle = OneCycleLR(optimizer, max_lr=1e-3,
                            steps_per_epoch=steps_per_epoch, epochs=epochs,
                            pct_start=0.3)  # 30% warmup, 70% decay

    # 4. Plateau — reduce LR when validation metric stagnates
    plateau = ReduceLROnPlateau(optimizer, mode='min', factor=0.5,
                                 patience=3, verbose=True)
    return cosine, sgdr, one_cycle, plateau


# ── Lambda scheduler: fully custom schedule ───────────────────────────
def custom_lambda_scheduler(optimizer, warmup=1000, total=50000):
    def lr_lambda(step):
        if step < warmup:
            return step / warmup
        progress = (step - warmup) / (total - warmup)
        return max(0.0, 0.5 * (1.0 + math.cos(math.pi * progress)))
    return torch.optim.lr_scheduler.LambdaLR(optimizer, lr_lambda)


# ── Training loop with scheduler ─────────────────────────────────────
def train_with_schedule(model, loader, epochs=5):
    optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3)
    scheduler = OneCycleLR(optimizer, max_lr=1e-3,
                            steps_per_epoch=len(loader), epochs=epochs)
    criterion = torch.nn.CrossEntropyLoss()
    for epoch in range(epochs):
        for Xb, yb in loader:
            optimizer.zero_grad()
            loss = criterion(model(Xb), yb)
            loss.backward()
            optimizer.step()
            scheduler.step()   # call after each batch for OneCycleLR
        cur_lr = scheduler.get_last_lr()[0]
        print(f"Epoch {epoch+1} | lr={cur_lr:.2e}")

Regularization — L1, L2 & Elastic Net

ML Analytics Generalization

Theory

Regularization penalizes model complexity in the loss function to prevent overfitting. L2 (Ridge) adds the sum of squared weights — it shrinks all weights uniformly toward zero, producing dense solutions. L1 (Lasso) adds the sum of absolute weights — it induces exact sparsity, driving unimportant weights to exactly zero. This makes L1 valuable for feature selection. Elastic Net combines both, balancing sparsity and stability. In neural networks, weight decay (AdamW's λ) is equivalent to L2 regularization.

Regularized Loss Functions
L2 (Ridge): L_reg = L_data + λ · Σ wᵢ²
L1 (Lasso): L_reg = L_data + λ · Σ |wᵢ|
Elastic Net: L_reg = L_data + λ₁·Σ|wᵢ| + λ₂·Σwᵢ²

Gradient: ∂L2/∂w = 2λw (smooth) | ∂L1/∂w = λ·sign(w) (sparse)

Real-World Usage

  • Credit scoring (logistic regression): L1 regularization selects the handful of significant features from 500+ financial indicators
  • Gene expression analysis: Lasso for feature selection across 20,000+ gene features to find cancer biomarkers
  • LLM weight decay (AdamW λ): L2 equivalent on transformer weights prevents memorization of training data
  • Ad click prediction (Google/Meta): L1 on billion-feature sparse ID embeddings to keep models deployable
regularization.py Python 
import torch
import torch.nn as nn
from sklearn.linear_model import Ridge, Lasso, ElasticNet
import numpy as np

# ── sklearn: Ridge / Lasso / ElasticNet ──────────────────────────────
def sklearn_regularization(X_train, y_train, X_test, y_test):
    models = {
        "Ridge (L2)":       Ridge(alpha=1.0),        # alpha = λ
        "Lasso (L1)":       Lasso(alpha=0.1),
        "ElasticNet":        ElasticNet(alpha=0.5, l1_ratio=0.5),
    }
    for name, model in models.items():
        model.fit(X_train, y_train)
        score = model.score(X_test, y_test)
        nonzero = np.sum(model.coef_ != 0)
        print(f"{name}: R²={score:.3f}, non-zero coefs={nonzero}")


# ── Manual L2 in PyTorch loss ─────────────────────────────────────────
def l2_regularized_loss(model, criterion, outputs, targets, lambda_=1e-4):
    base_loss = criterion(outputs, targets)
    l2_penalty = 0.0
    for param in model.parameters():
        l2_penalty += torch.sum(param ** 2)
    return base_loss + lambda_ * l2_penalty


# ── Manual L1 (sparsity-inducing) ─────────────────────────────────────
def l1_regularized_loss(model, criterion, outputs, targets, lambda_=1e-5):
    base_loss  = criterion(outputs, targets)
    l1_penalty = sum(torch.sum(torch.abs(p)) for p in model.parameters())
    return base_loss + lambda_ * l1_penalty


# ── Full neural-net training with L2 regularization ───────────────────
class RegularizedNet(nn.Module):
    def __init__(self, in_dim, hidden, out_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, out_dim)
        )
    def forward(self, x): return self.net(x)


def train_with_l2(X, y, lambda_=1e-4, epochs=50):
    model     = RegularizedNet(10, 64, 1)
    # weight_decay in AdamW IS L2 regularization (properly decoupled)
    optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3,
                                   weight_decay=lambda_)
    criterion = nn.MSELoss()
    for _ in range(epochs):
        optimizer.zero_grad()
        loss = criterion(model(X), y)
        loss.backward()
        optimizer.step()
    return model

Batch Normalization & Layer Norm

Deep Learning Transformers Training Stability

Theory

Internal covariate shift — the change in distribution of layer inputs during training — slows convergence. Batch Normalization normalizes each feature across the batch to zero mean and unit variance, then applies learnable scale γ and shift β. This smooths the loss landscape, allowing much larger learning rates. Layer Normalization normalizes across the feature dimension for a single sample, making it ideal for transformers where batch statistics are unstable due to variable-length sequences. RMS Norm (used in LLaMA) drops the mean subtraction for efficiency.

Batch Norm & Layer Norm Equations
BatchNorm: μ_B = (1/m)Σxᵢ, σ²_B = (1/m)Σ(xᵢ−μ_B)²
x̂ᵢ = (xᵢ − μ_B) / √(σ²_B + ε), yᵢ = γ·x̂ᵢ + β

LayerNorm: μ = mean(x), σ² = var(x) [across features, per sample]
x̂ = (x − μ) / √(σ² + ε), y = γ·x̂ + β

RMSNorm: RMS(x) = √((1/n)Σxᵢ²), y = x / RMS(x) · γ

Real-World Usage

  • ResNet, EfficientNet (CV): BatchNorm after every conv layer enables training 100+ layer networks
  • GPT-2 / GPT-4: Pre-LayerNorm (before attention) for stable gradient flow in deep transformers
  • LLaMA / Mistral: RMSNorm — same stability gains as LayerNorm with 15% fewer computations
  • Anomaly detection systems: BatchNorm's running statistics catch distribution shift in production data
normalization.py Python 
import torch
import torch.nn as nn
import numpy as np

# ── Batch Norm from scratch ───────────────────────────────────────────
class BatchNorm1D(nn.Module):
    def __init__(self, num_features, eps=1e-5, momentum=0.1):
        super().__init__()
        self.eps      = eps
        self.momentum = momentum
        self.gamma    = nn.Parameter(torch.ones(num_features))   # scale
        self.beta     = nn.Parameter(torch.zeros(num_features))  # shift
        self.register_buffer('running_mean', torch.zeros(num_features))
        self.register_buffer('running_var',  torch.ones(num_features))

    def forward(self, x):
        if self.training:
            mu   = x.mean(dim=0)        # mean across batch
            var  = x.var(dim=0, unbiased=False)
            self.running_mean = (1 - self.momentum) * self.running_mean \
                                 + self.momentum * mu.detach()
            self.running_var  = (1 - self.momentum) * self.running_var \
                                 + self.momentum * var.detach()
        else:
            mu, var = self.running_mean, self.running_var
        x_hat = (x - mu) / torch.sqrt(var + self.eps)
        return self.gamma * x_hat + self.beta


# ── RMSNorm (LLaMA-style) ─────────────────────────────────────────────
class RMSNorm(nn.Module):
    def __init__(self, dim, eps=1e-6):
        super().__init__()
        self.eps   = eps
        self.scale = nn.Parameter(torch.ones(dim))

    def forward(self, x):
        rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)
        return self.scale * (x / rms)


# ── Comparison in a transformer block ────────────────────────────────
class TransformerBlock(nn.Module):
    def __init__(self, d_model=512, norm="layer"):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, num_heads=8, batch_first=True)
        self.ff   = nn.Sequential(nn.Linear(d_model, d_model*4), nn.GELU(),
                                    nn.Linear(d_model*4, d_model))
        NormClass = RMSNorm if norm == "rms" else nn.LayerNorm
        self.norm1 = NormClass(d_model)
        self.norm2 = NormClass(d_model)

    def forward(self, x):
        # Pre-norm (modern standard: norm before sub-layer)
        x = x + self.attn(self.norm1(x), self.norm1(x), self.norm1(x))[0]
        x = x + self.ff(self.norm2(x))
        return x

Dropout & Variants

Regularization LLM Generalization

Theory

Dropout randomly zeroes activations with probability p during training, preventing co-adaptation of neurons. Each forward pass trains a different sub-network; at inference the full network runs with weights scaled by (1−p). This ensemble effect dramatically reduces overfitting. Variants include DropConnect (zero weights, not activations), Spatial Dropout (whole feature maps in CNNs), and Attention Dropout (in transformer attention matrices). Monte Carlo Dropout keeps dropout active at inference for Bayesian uncertainty estimation — used in safety-critical ML systems.

Dropout Mechanism
Training: h̃ = h ⊙ m, where mᵢ ~ Bernoulli(1−p) [mask]
Inference: h̃ = (1−p) · h [scale down]
Or equivalently — Inverted Dropout (PyTorch default):
Training: h̃ = (h ⊙ m) / (1−p) [scale up during train]
Inference: h̃ = h [no scaling needed]

Real-World Usage

  • BERT: p=0.1 dropout on attention weights and feed-forward layers; critical for fine-tuning stability
  • Medical imaging AI (uncertainty): MC Dropout at inference to quantify prediction uncertainty for radiologist review
  • Tabular ML (churn prediction): Dropout in MLPs prevents memorizing customer IDs
  • LoRA fine-tuning: Dropout on LoRA adapter layers (rank decomposition) prevents adapter overfitting
dropout.py Python 
import torch
import torch.nn as nn
import numpy as np

# ── Inverted Dropout from scratch ─────────────────────────────────────
def dropout(x, p=0.5, training=True):
    if not training or p == 0: return x
    mask = (torch.rand_like(x) > p).float()
    return x * mask / (1.0 - p)   # inverted: scale up during train


# ── Deep MLP with dropout at each hidden layer ────────────────────────
class MLPWithDropout(nn.Module):
    def __init__(self, in_dim, hidden_dims, out_dim, drop_p=0.3):
        super().__init__()
        layers = []
        prev   = in_dim
        for h in hidden_dims:
            layers += [nn.Linear(prev, h), nn.BatchNorm1d(h),
                       nn.ReLU(), nn.Dropout(p=drop_p)]
            prev = h
        layers.append(nn.Linear(prev, out_dim))
        self.net = nn.Sequential(*layers)

    def forward(self, x): return self.net(x)


# ── Monte Carlo Dropout — uncertainty at inference ─────────────────────
class MCDropoutModel(nn.Module):
    def __init__(self, in_dim, out_dim, p=0.2):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_dim, 128), nn.ReLU(), nn.Dropout(p),
            nn.Linear(128, 64),   nn.ReLU(), nn.Dropout(p),
            nn.Linear(64, out_dim)
        )

    def forward(self, x): return self.net(x)

    def predict_with_uncertainty(self, x, n_samples=100):
        self.train()  # keep dropout active at inference!
        preds = torch.stack([self.net(x) for _ in range(n_samples)])
        mean  = preds.mean(dim=0)
        std   = preds.std(dim=0)   # epistemic uncertainty
        return mean, std


# ── Attention Dropout (used in transformer self-attention) ────────────
class AttentionWithDropout(nn.Module):
    def __init__(self, d_model, n_heads, attn_drop=0.1):
        super().__init__()
        self.attn    = nn.MultiheadAttention(d_model, n_heads, dropout=attn_drop,
                                              batch_first=True)
        self.proj_drop = nn.Dropout(attn_drop)

    def forward(self, x):
        out, weights = self.attn(x, x, x)
        return self.proj_drop(out), weights

Loss Functions

ML LLM Analytics

Theory

The loss function quantifies how wrong predictions are — it is the function we minimize. Choosing the right loss is as important as the architecture. MSE (regression) assumes Gaussian noise, penalizes outliers heavily. MAE (Huber) is robust to outliers. Cross-Entropy (classification) maximizes log-likelihood of true class probabilities. KL Divergence measures how much distribution Q diverges from reference P — fundamental to variational autoencoders and RLHF KL penalties. Contrastive losses (NT-Xent, InfoNCE) are used in self-supervised learning and retrieval models.

Key Loss Equations
MSE: L = (1/n) Σ (ŷᵢ − yᵢ)²
MAE: L = (1/n) Σ |ŷᵢ − yᵢ|
BCE: L = −(1/n) Σ [yᵢ log(ŷᵢ) + (1−yᵢ)log(1−ŷᵢ)]
Cross-Entropy: L = −(1/n) Σ Σ yᵢⱼ · log(ŷᵢⱼ) [multi-class]
KL Divergence: KL(P‖Q) = Σ P(x) log(P(x)/Q(x))
InfoNCE: L = −log[ exp(sim(z,z⁺)/τ) / Σₖ exp(sim(z,zₖ)/τ) ]

Real-World Usage

  • LLM next-token prediction: Cross-entropy over vocabulary (50k tokens) — the fundamental pre-training objective
  • RLHF (ChatGPT): KL penalty term keeps fine-tuned model close to reference policy, preventing reward hacking
  • Sentence-BERT / retrieval embeddings: InfoNCE/NT-Xent on positive/negative pairs for semantic search
  • Medical regression (dosage prediction): Huber loss handles outlier lab values gracefully
  • Fraud detection (imbalanced): Focal Loss down-weights easy negatives, focuses training on hard fraud cases

# ── Focal Loss — for class imbalance (fraud, rare disease) ────────────
class FocalLoss(nn.Module):
    def __init__(self, gamma=2.0, alpha=0.25):
        super().__init__()
        self.gamma = gamma   # focusing parameter (2 = standard)
        self.alpha = alpha   # class weight for positives

    def forward(self, logits, targets):
        bce_loss = F.binary_cross_entropy_with_logits(logits, targets, reduction='none')
        p_t      = torch.exp(-bce_loss)             # estimated probability
        focal    = (1 - p_t) ** self.gamma * bce_loss
        alpha_t  = self.alpha * targets + (1 - self.alpha) * (1 - targets)
        return (alpha_t * focal).mean()


# ── InfoNCE / NT-Xent — contrastive loss for embeddings ──────────────
def infonce_loss(z1, z2, temperature=0.07):
    """z1, z2: L2-normalized embeddings [B, D] — positives are row-pairs"""
    z1 = F.normalize(z1, dim=-1)
    z2 = F.normalize(z2, dim=-1)
    B  = z1.shape[0]
    sim = torch.mm(z1, z2.T) / temperature   # [B, B] similarity matrix
    labels = torch.arange(B)                 # diagonal = positive pairs
    loss_12 = F.cross_entropy(sim, labels)
    loss_21 = F.cross_entropy(sim.T, labels)
    return (loss_12 + loss_21) / 2.0


# ── KL penalty for RLHF — keeping policy near reference ──────────────
def rlhf_kl_penalty(log_p_new, log_p_ref, beta=0.1):
    """KL(π_new ‖ π_ref) — prevents reward hacking in RLHF"""
    kl = (log_p_new.exp() * (log_p_new - log_p_ref)).sum(dim=-1).mean()
    return beta * kl

Momentum & Nesterov Accelerated Gradient

Optimization Convergence

Theory

Vanilla SGD oscillates in ravines — high curvature in one direction, low in another. Momentum accumulates a velocity vector in the gradient direction, dampening oscillations and accelerating progress in persistent directions (like a ball rolling downhill). Nesterov Accelerated Gradient (NAG) evaluates the gradient at the "lookahead" position (after applying current velocity), giving a corrective signal that reduces overshooting. NAG converges at O(1/t²) vs SGD's O(1/t) for convex functions.

Momentum & NAG Equations
Momentum: v_t = γ·v_(t-1) + η·∇L(θ_t)
θ_(t+1) = θ_t − v_t

Nesterov: v_t = γ·v_(t-1) + η·∇L(θ_t − γ·v_(t-1)) ← gradient at lookahead
θ_(t+1) = θ_t − v_t

Typical: γ = 0.9 (momentum coefficient)

Real-World Usage

  • ResNet / EfficientNet training: SGD + Nesterov momentum (γ=0.9) outperforms Adam for image classification final accuracy
  • Physics-based simulation (scientific ML): NAG's convergence guarantees matter for stable PDE solvers
  • Recommendation system embedding tables: Momentum SGD for sparse gradient updates on categorical features
momentum.py Python 
import numpy as np
import torch

# ── Momentum SGD from scratch ─────────────────────────────────────────
class MomentumSGD:
    def __init__(self, params, lr=0.01, momentum=0.9):
        self.params   = params
        self.lr       = lr
        self.gamma    = momentum
        self.velocity = [np.zeros_like(p) for p in params]

    def step(self, grads):
        for i, (p, g) in enumerate(zip(self.params, grads)):
            self.velocity[i] = self.gamma * self.velocity[i] + self.lr * g
            p -= self.velocity[i]


# ── Nesterov from scratch ─────────────────────────────────────────────
class NesterovSGD:
    def __init__(self, params, lr=0.01, momentum=0.9):
        self.params   = params
        self.lr       = lr
        self.gamma    = momentum
        self.velocity = [np.zeros_like(p) for p in params]

    def step(self, grad_fn):
        # evaluate gradient at lookahead position
        lookaheads = [p - self.gamma * v
                      for p, v in zip(self.params, self.velocity)]
        grads = grad_fn(lookaheads)
        for i, (p, g) in enumerate(zip(self.params, grads)):
            self.velocity[i] = self.gamma * self.velocity[i] + self.lr * g
            p -= self.velocity[i]


# ── PyTorch: SGD with Nesterov ────────────────────────────────────────
def build_nesterov_optimizer(model, lr=0.05, momentum=0.9, wd=1e-4):
    """Standard recipe for ImageNet training (He et al., ResNet paper)"""
    return torch.optim.SGD(
        model.parameters(), lr=lr, momentum=momentum,
        nesterov=True, weight_decay=wd
    )

Quantization for LLM Inference

LLM Serving Efficiency Production

Theory

Full-precision (FP32) weights consume 4 bytes each — a 70B-parameter model needs 280GB of VRAM. Quantization represents weights in lower precision. INT8 halves memory; INT4 (GGUF, GPTQ, AWQ) cuts to 25%. The core trade-off is between accuracy loss and memory/speed gain. Post-Training Quantization (PTQ) requires only a small calibration set. GPTQ uses second-order Hessian information to minimize quantization error layer by layer. AWQ (Activation-Aware Quantization) identifies and protects salient weights that most affect accuracy, outperforming GPTQ on many benchmarks.

Quantization Formula
Symmetric INT8: x_q = round(x / s), s = max(|x|) / 127
Dequantize: x̂ = x_q · s

GPTQ objective: argmin_Ŵ ‖(W − Ŵ)X‖²₂ (per-layer, Hessian-guided)

Memory savings: FP32→INT8: 4x | FP32→INT4: 8x | FP16→INT4: 4x

Real-World Usage

  • llama.cpp / Ollama: Q4_K_M quantization runs LLaMA-3 70B on a consumer 48GB GPU at 20 tok/s
  • vLLM production serving: AWQ INT4 halves cost per-token on cloud GPU instances (A100/H100)
  • Mobile LLM (Phi-3-mini): INT4 quantization enables on-device inference on iPhone 15 Pro
  • Analytics ML inference: INT8 quantized XGBoost/neural net for real-time fraud scoring (<5ms latency)

Weight Pruning & Sparsity

Model Compression LLM Efficiency

Theory

The Lottery Ticket Hypothesis (Frankle & Carlin 2019) showed that large networks contain small sparse sub-networks ("winning tickets") that train just as well. Pruning removes weights by magnitude (small weights → zero), structured channel pruning removes entire neurons/heads, and SparseGPT can prune 50% of an LLM's weights in one shot without retraining. Sparsity improves inference speed on hardware that supports it (NVIDIA sparse tensor cores) and reduces memory. L0 regularization directly minimizes the count of non-zero parameters.

Magnitude-Based Pruning Criterion
Global threshold: prune wᵢⱼ if |wᵢⱼ| < percentile(|W|, p%)

L0 regularization: L = L_data + λ · ‖W‖₀ (‖·‖₀ = count of non-zeros)

Taylor expansion criterion (structured): score(fᵢ) = |δL/δfᵢ · fᵢ|²

Real-World Usage

  • SparseGPT (Meta): 50% sparsity on OPT-175B with <1% perplexity increase, no retraining
  • Attention head pruning (BERT): Remove 30-40% of attention heads with minimal NLP benchmark drop
  • Mobile CNN pruning: Channel pruning reduces MobileNet FLOPs by 50% for edge devices
  • Real-time analytics scoring: Pruned gradient boosted trees for sub-millisecond inference in ad bidding
pruning.py Python 
import torch
import torch.nn as nn
import torch.nn.utils.prune as prune

# ── Unstructured magnitude pruning ────────────────────────────────────
def apply_global_unstructured_pruning(model, amount=0.3):
    """Remove 30% of weights globally by smallest absolute magnitude"""
    params_to_prune = [
        (module, "weight")
        for name, module in model.named_modules()
        if isinstance(module, nn.Linear)
    ]
    prune.global_unstructured(
        params_to_prune,
        pruning_method=prune.L1Unstructured,
        amount=amount,
    )
    return model


# ── Structured (channel/neuron) pruning ──────────────────────────────
def apply_structured_pruning(model, amount=0.2):
    """Remove 20% of neurons (entire rows) per Linear layer"""
    for name, module in model.named_modules():
        if isinstance(module, nn.Linear):
            prune.ln_structured(module, name="weight", amount=amount,
                                n=2, dim=0)  # L2 norm, dim=0=output neurons
    return model


# ── Iterative magnitude pruning with fine-tuning ──────────────────────
def iterative_pruning(model, train_loader, target_sparsity=0.9,
                       steps=5, finetune_epochs=2):
    amount_per_step = 1 - (1 - target_sparsity) ** (1 / steps)
    optimizer  = torch.optim.AdamW(model.parameters(), lr=5e-5)
    criterion  = nn.CrossEntropyLoss()

    for step in range(steps):
        # prune
        apply_global_unstructured_pruning(model, amount=amount_per_step)
        sparsity = compute_sparsity(model)
        print(f"Step {step+1}: sparsity={sparsity:.1%}")
        # fine-tune to recover accuracy
        model.train()
        for _ in range(finetune_epochs):
            for Xb, yb in train_loader:
                optimizer.zero_grad()
                criterion(model(Xb), yb).backward()
                optimizer.step()

    # make pruning permanent (remove mask buffers)
    for module in model.modules():
        if isinstance(module, nn.Linear):
            try: prune.remove(module, 'weight')
            except: pass
    return model


def compute_sparsity(model):
    zeros = total = 0
    for p in model.parameters():
        zeros += (p == 0).sum().item()
        total += p.numel()
    return zeros / total

RAG & Embedding Optimization

LLM / RAG Search Retrieval

Theory

Retrieval-Augmented Generation grounds LLM outputs in external knowledge. The retrieval bottleneck determines answer quality: we embed documents and queries into a dense vector space, then find nearest neighbors. Key optimizations: (1) Chunking strategy — chunk size, overlap, and semantic splitting affect recall. (2) Embedding model choice — domain-specific fine-tuned models (e.g., BGE, E5) outperform general models. (3) ANN indexing — FAISS HNSW and IVF indices allow sub-millisecond search over 100M+ vectors. (4) Reranking — a cross-encoder reranker (ColBERT, BGE-reranker) re-scores top-k candidates using full attention, dramatically improving precision.

Similarity & Retrieval Metrics
Cosine similarity: sim(q,d) = (q·d) / (‖q‖·‖d‖)
Dot product: sim(q,d) = q·d (requires normalized embeddings)
MMR (diversity): MMR = argmax_dᵢ [λ·sim(q,dᵢ) − (1−λ)·max_dⱼ sim(dᵢ,dⱼ)]

HNSW search: O(log N) approximate nearest neighbor lookup

Real-World Usage

  • Enterprise RAG (legal/medical docs): FAISS HNSW + BGE-reranker for <200ms retrieval over 10M document chunks
  • GitHub Copilot context retrieval: Embedding-based code search to surface relevant functions into LLM context
  • Customer support bots: RAG over FAQ knowledge base reduces hallucination rate from ~40% to <5%
  • Analytics query routing: Embed user NL queries to route to correct SQL template via semantic similarity
rag_optimization.py Python 
import numpy as np
from sentence_transformers import SentenceTransformer, CrossEncoder
import faiss

# ── Build FAISS HNSW index for fast ANN retrieval ─────────────────────
def build_faiss_index(embeddings: np.ndarray, use_hnsw=True):
    dim = embeddings.shape[1]
    if use_hnsw:
        # HNSW: O(log N) lookup, great recall, no training needed
        index = faiss.IndexHNSWFlat(dim, 32)  # 32 = M (graph degree)
        index.hnsw.efConstruction = 200        # higher = better recall, slower build
        index.hnsw.efSearch       = 64         # search-time recall vs speed
    else:
        # IVF: train k-means centroids, then search relevant clusters
        nlist = min(4096, len(embeddings) // 40)
        quantizer = faiss.IndexFlatIP(dim)
        index     = faiss.IndexIVFFlat(quantizer, dim, nlist)
        index.train(embeddings)
        index.nprobe = 64          # check 64 clusters per query

    faiss.normalize_L2(embeddings)
    index.add(embeddings)
    return index


# ── Full RAG pipeline with reranking ──────────────────────────────────
class OptimizedRAG:
    def __init__(self, docs, embed_model="BAAI/bge-base-en-v1.5",
                 reranker_model="BAAI/bge-reranker-v2-m3"):
        self.docs     = docs
        self.embedder = SentenceTransformer(embed_model)
        self.reranker = CrossEncoder(reranker_model)
        self.index    = None
        self._build_index()

    def _build_index(self):
        embs = self.embedder.encode(self.docs, normalize_embeddings=True,
                                    show_progress_bar=True,
                                    batch_size=256)
        self.embs  = embs.astype('float32')
        self.index = build_faiss_index(self.embs)

    def retrieve(self, query, top_k=20, rerank_top=5):
        # 1. Bi-encoder retrieval (fast, recall-focused)
        q_emb = self.embedder.encode([query], normalize_embeddings=True)
        q_emb = q_emb.astype('float32')
        D, I  = self.index.search(q_emb, top_k)
        candidates = [self.docs[i] for i in I[0] if i != -1]

        # 2. Cross-encoder reranking (slow, precision-focused)
        pairs   = [(query, doc) for doc in candidates]
        scores  = self.reranker.predict(pairs)
        ranked  = sorted(zip(scores, candidates), reverse=True)[:rerank_top]
        return [doc for _, doc in ranked]


# ── Chunking strategies ───────────────────────────────────────────────
def fixed_chunk(text, size=512, overlap=50):
    tokens = text.split()
    chunks = []
    for i in range(0, len(tokens), size - overlap):
        chunks.append(" ".join(tokens[i : i + size]))
    return chunks

def semantic_chunk(text, embed_model, threshold=0.85):
    """Split on semantic discontinuities — better for mixed-topic docs"""
    sentences = text.split(".")
    embs = embed_model.encode(sentences, normalize_embeddings=True)
    chunks, buf = [], [sentences[0]]
    for i in range(1, len(sentences)):
        sim = np.dot(embs[i-1], embs[i])
        if sim < threshold:   # semantic break detected
            chunks.append(" ".join(buf)); buf = []
        buf.append(sentences[i])
    if buf: chunks.append(" ".join(buf))
    return chunks

Hyperparameter Optimization

AutoML Analytics Bayesian Opt.

Theory

Hyperparameters (learning rate, batch size, architecture dimensions, regularization strength) control the training process itself — they cannot be optimized by gradient descent. Grid Search exhaustively tries all combinations (exponential in dimension). Random Search (Bergstra & Bengio 2012) is often just as good with far fewer evaluations. Bayesian Optimization fits a surrogate model (Gaussian Process or Tree Parzen Estimator) over the hyperparameter → performance mapping, then uses an acquisition function (Expected Improvement, UCB) to propose the most informative next trial. Hyperband combines early stopping with bracket-based resource allocation for extreme efficiency.

Bayesian Optimization — Expected Improvement
EI(x) = E[max(f(x) − f*, 0)]

For GP surrogate with mean μ(x), std σ(x):
EI(x) = (μ(x) − f*) · Φ(Z) + σ(x) · φ(Z)
Z = (μ(x) − f*) / σ(x)

where Φ = CDF, φ = PDF of standard normal, f* = best observed value

Real-World Usage

  • LLM training hyperparams (DeepMind Chinchilla): Bayesian opt to find optimal compute-efficient lr × tokens × params
  • Kaggle competition ML: Optuna TPE to tune XGBoost/LightGBM over 100+ trials for leaderboard wins
  • Neural architecture search (NAS): Hyperband + BO to find optimal transformer depth/width tradeoff
  • A/B tested recommendation systems: Random search over embedding dim, learning rate, negative sampling strategy

# ── Optuna for PyTorch neural network ─────────────────────────────────
def nn_objective(trial, X_train, y_train, X_val, y_val):
    lr          = trial.suggest_float("lr", 1e-5, 1e-2, log=True)
    n_layers    = trial.suggest_int("n_layers", 1, 5)
    hidden_dim  = trial.suggest_categorical("hidden", [64, 128, 256, 512])
    dropout_p   = trial.suggest_float("dropout", 0.0, 0.5)
    weight_dec  = trial.suggest_float("wd", 1e-6, 1e-2, log=True)

    layers = []
    in_dim = X_train.shape[1]
    for _ in range(n_layers):
        layers += [nn.Linear(in_dim, hidden_dim), nn.ReLU(), nn.Dropout(dropout_p)]
        in_dim = hidden_dim
    layers.append(nn.Linear(in_dim, 1))
    model     = nn.Sequential(*layers)
    optimizer = torch.optim.AdamW(model.parameters(), lr=lr, weight_decay=weight_dec)
    criterion = nn.BCEWithLogitsLoss()

    for _ in range(20):   # quick training for HPO
        model.train()
        optimizer.zero_grad()
        loss = criterion(model(X_train).squeeze(), y_train)
        loss.backward(); optimizer.step()

    model.eval()
    with torch.no_grad():
        val_loss = criterion(model(X_val).squeeze(), y_val).item()
    return val_loss  # minimize val loss


# ── Population-Based Training (PBT) concept ───────────────────────────
def pbt_exploit_explore(population, metric_fn, perturb=0.2):
    """Simplified PBT: copy top performers' params and perturb LR"""
    scores  = [(metric_fn(agent), agent) for agent in population]
    scores.sort(key=lambda x: x[0], reverse=True)
    top_half = [a for _, a in scores[:len(scores)//2]]
    bottom   = [a for _, a in scores[len(scores)//2:]]
    import copy, random
    for agent in bottom:
        parent = random.choice(top_half)
        agent['state'] = copy.deepcopy(parent['state'])  # exploit
        agent['lr']    = parent['lr'] * random.uniform(1-perturb, 1+perturb)  # explore
    return population
