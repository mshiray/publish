01

Functions of Multiple Variables The foundation of all ML loss landscapes

Theory

A function of multiple variables maps a vector input x ∈ ℝⁿ to a scalar (or vector) output. In ML, almost every quantity of interest — loss, accuracy, log-likelihood — is a function of many parameters simultaneously.

The domain is a subset of ℝⁿ. The function defines a hyper-surface (loss landscape) in ℝⁿ⁺¹ that we navigate during training.

Key Formulas
f : ℝⁿ → ℝ, f(x₁, x₂, …, xₙ)

-- MSE loss (n parameters, m samples): L(θ) = (1/m) Σᵢ (yᵢ - ŷᵢ(θ))²

-- Cross-entropy loss: L(θ) = -(1/m) Σᵢ [yᵢ log(σ(θᵀxᵢ)) + (1-yᵢ) log(1 - σ(θᵀxᵢ))]
Python Code
python 
import numpy as np
import matplotlib.pyplot as plt
from mpl_toolkits.mplot3d import Axes3D

# ── Define a 2-variable loss-like function ──────────────────────
def loss_surface(w1, w2):
    """Quadratic bowl — typical convex loss shape."""
    return w1**2 + 2*w2**2 + 0.5*w1*w2

02

Partial Derivatives How output changes w.r.t. one parameter at a time

Theory

The partial derivative ∂f/∂xᵢ measures the instantaneous rate of change of f when only xᵢ varies and all other variables are held constant. It answers: "if I nudge weight wᵢ by ε, how much does the loss change?"

Higher-order partials (∂²f/∂xᵢ∂xⱼ) encode curvature and interaction between parameters — crucial for second-order optimizers.

Key Formulas
-- Definition (limit form): ∂f/∂xᵢ = lim[h→0] [f(x + h·eᵢ) - f(x)] / h

-- Example: f(w₁,w₂) = w₁²·w₂ + e^(w₂) ∂f/∂w₁ = 2w₁·w₂ ∂f/∂w₂ = w₁² + e^(w₂)

-- Second-order (mixed): ∂²f/∂w₁∂w₂ = 2w₁ (= ∂²f/∂w₂∂w₁ by Clairaut's theorem)
Python Code
python 
import sympy as sp
import numpy as np
import torch

# ── Symbolic partial derivatives with SymPy ──────────────────────
w1, w2 = sp.symbols('w1 w2')
f = w1**2 * w2 + sp.exp(w2)

df_dw1 = sp.diff(f, w1)
df_dw2 = sp.diff(f, w2)
d2f    = sp.diff(f, w1, w2)           # mixed partial

print(f"∂f/∂w1 = {df_dw1}")           # 2*w1*w2
print(f"∂f/∂w2 = {df_dw2}")           # w1**2 + exp(w2)
print(f"∂²f/∂w1∂w2 = {d2f}")         # 2*w1

# ── Numerical partial derivative (finite differences) ────────────
def partial_fd(func, x, idx, h=1e-5):
    """Finite-difference partial derivative w.r.t. x[idx]."""
    x_fwd = x.copy(); x_fwd[idx] += h
    x_bwd = x.copy(); x_bwd[idx] -= h
    return (func(x_fwd) - func(x_bwd)) / (2 * h)

def mse_loss(theta, X, y):
    preds = X @ theta
    return np.mean((y - preds)**2)

# Toy dataset
np.random.seed(42)
X     = np.random.randn(100, 3)
theta = np.array([1.0, -0.5, 2.0])
y     = X @ theta + np.random.randn(100) * 0.1

for i in range(3):
    pd = partial_fd(lambda t: mse_loss(t, X, y), theta, i)
    print(f"∂L/∂θ{i} ≈ {pd:.6f}")

# ── Autograd via PyTorch ─────────────────────────────────────────
w = torch.tensor([1.0, 2.0], requires_grad=True)
f_torch = w[0]**2 * w[1] + torch.exp(w[1])
f_torch.backward()
print(f"∂f/∂w via autograd: {w.grad}")   # [2*1*2,  1²+e²]

03

Gradient Vector Direction of steepest ascent in parameter space

Theory

The gradient ∇f collects all partial derivatives into a single vector. It points in the direction of steepest increase of f. In optimization, we move against the gradient (gradient descent) to minimize loss.

Geometrically, ∇f(x₀) is perpendicular to the level set (contour) passing through x₀ — a key insight for understanding why gradient descent moves orthogonally to iso-loss contours.

Key Formulas
∇f = [∂f/∂x₁, ∂f/∂x₂, …, ∂f/∂xₙ]ᵀ ∈ ℝⁿ

-- Gradient of MSE loss L(θ) = (1/m)||y - Xθ||²: ∇_θ L = (-2/m) Xᵀ(y - Xθ)

-- Gradient of Cross-Entropy (logistic): ∇_θ L = (1/m) Xᵀ(σ(Xθ) - y)

-- Gradient magnitude = slope steepness: ||∇f|| = sqrt(Σᵢ (∂f/∂xᵢ)²)
Python Code
python 
import numpy as np
import matplotlib.pyplot as plt

# ── Analytical gradient of MSE ────────────────────────────────────
def gradient_mse(X, y, theta):
    m = len(y)
    residuals = y - X @ theta
    return (-2/m) * X.T @ residuals     # ∇L ∈ ℝⁿ

np.random.seed(0)
X_aug = np.column_stack([np.ones(80), np.random.randn(80, 2)])
true_w = np.array([1.0, 2.0, -1.5])
y_data = X_aug @ true_w + np.random.randn(80) * 0.3

theta_init = np.zeros(3)
g = gradient_mse(X_aug, y_data, theta_init)
print(f"Gradient at θ=0: {g}")
print(f"Gradient norm:   {np.linalg.norm(g):.4f}")

04

Directional Derivatives Rate of change along any arbitrary direction

Theory

The directional derivative D_û f(x) measures how fast f changes when moving in a specific unit direction û. It generalizes the partial derivative (which is D_eᵢ f). It equals the dot product of the gradient with û: the gradient is the direction that maximizes this rate.

Key Formulas
D_û f(x) = ∇f(x) · û = ||∇f|| cos(θ)

-- Maximum rate of increase: direction = ∇f/||∇f||, value = ||∇f|| -- Maximum rate of decrease: direction = -∇f/||∇f|| -- Zero change: û ⊥ ∇f (moving along a contour)
Python Code
python 
import numpy as np

def f(x):  return x[0]**2 + 2*x[1]**2 + x[0]*x[1]

def grad_f(x):
    return np.array([2*x[0] + x[1],  4*x[1] + x[0]])

x0  = np.array([1.0, 2.0])
g   = grad_f(x0)

# Arbitrary direction
v   = np.array([1.0, 1.0])
u   = v / np.linalg.norm(v)           # normalise to unit vector

D_u = np.dot(g, u)
print(f"Directional derivative along u: {D_u:.4f}")
print(f"Max rate (||∇f||):              {np.linalg.norm(g):.4f}")
print(f"Steepest ascent direction:      {g/np.linalg.norm(g)}")

# ── Application: adaptive learning rate intuition ────────────────
# Different directions have different curvatures → motivates Adagrad/Adam
angles   = np.linspace(0, 2*np.pi, 360)
dir_devs = [np.dot(g, np.array([np.cos(a), np.sin(a)])) for a in angles]
print(f"Max directional derivative: {max(dir_devs):.4f}")
print(f"Min directional derivative: {min(dir_devs):.4f}")

05

Multivariate Chain Rule Gradient flow through composed functions

Theory

When f = g ∘ h (a composition), the chain rule tells us how to differentiate f. In the multivariate case, if z = g(y) and y = h(x), then the total derivative is a matrix product of Jacobians. This is the mathematical engine behind backpropagation.

Key Formulas
-- Scalar chain rule: dz/dx = (dz/dy)(dy/dx)

-- Multivariate (tree rule): ∂z/∂xᵢ = Σⱼ (∂z/∂yⱼ)(∂yⱼ/∂xᵢ)

-- Matrix form (Jacobian product): ∂z/∂x = (∂z/∂y)(∂y/∂x) = J_g · J_h

-- Neural network layer-by-layer: ∂L/∂W⁽ˡ⁾ = (∂L/∂a⁽ˡ⁾)(∂a⁽ˡ⁾/∂z⁽ˡ⁾)(∂z⁽ˡ⁾/∂W⁽ˡ⁾)
Python Code
python 
import numpy as np
import torch
import torch.nn as nn

# ── Manual chain rule: f(g(x)) = sin(x² + 1) ─────────────────────
x_val = 2.0
g     = x_val**2 + 1         # g(x) = x² + 1
dg_dx = 2 * x_val             # dg/dx
df_dg = np.cos(g)              # df/dg = cos(g)
df_dx = df_dg * dg_dx          # chain rule
print(f"d/dx sin(x²+1) at x=2: {df_dx:.6f}")

# ── Verify with torch ─────────────────────────────────────────────
x = torch.tensor(2.0, requires_grad=True)
z = torch.sin(x**2 + 1)
z.backward()
print(f"Torch autograd result:  {x.grad.item():.6f}")

# ── 2-layer network: manual chain rule ───────────────────────────
def relu(z): return np.maximum(0, z)
def relu_prime(z): return (z > 0).astype(float)
def sigmoid(z): return 1 / (1 + np.exp(-z))

np.random.seed(1)
x_in  = np.random.randn(4)       # input features
W1    = np.random.randn(3, 4)    # Layer 1 weights
W2    = np.random.randn(1, 3)    # Layer 2 weights
y_t   = np.array([1.0])

# Forward pass
z1 = W1 @ x_in;      a1 = relu(z1)
z2 = W2 @ a1;        a2 = sigmoid(z2)
L  = -y_t * np.log(a2 + 1e-8)

# Backward pass (chain rule)
dL_da2 = -y_t / (a2 + 1e-8)
da2_dz2 = a2 * (1 - a2)                    # sigmoid derivative
dz2_dW2 = a1                                 # ∂z2/∂W2
dL_dW2 = (dL_da2 * da2_dz2)[:, None] * dz2_dW2[None, :]

dL_da1 = W2.T @ (dL_da2 * da2_dz2)
dL_dz1 = dL_da1 * relu_prime(z1)
dL_dW1 = dL_dz1[:, None] * x_in[None, :]

print(f"∂L/∂W2 shape: {dL_dW2.shape}")
print(f"∂L/∂W1 shape: {dL_dW1.shape}")

06

Jacobian Matrix Full derivative of vector-valued functions

Theory

When a function maps ℝⁿ → ℝᵐ, the derivative is an m×n matrix of all partial derivatives — the Jacobian. Each row is the gradient of one output component. The Jacobian generalizes the scalar derivative to vector functions and underlies both forward- and reverse-mode automatic differentiation.

The determinant of the Jacobian (Jacobian determinant) measures volume change under the transformation — critical in normalizing flows and change-of-variables in probabilistic models.

Key Formulas
-- f: ℝⁿ → ℝᵐ, J ∈ ℝᵐˣⁿ J_f(x) = [∂fᵢ/∂xⱼ] (i=1..m, j=1..n)

-- Softmax Jacobian (important in classification): ∂softmax(z)ᵢ/∂zⱼ = sᵢ(δᵢⱼ - sⱼ)

-- Change of variables (normalizing flows): p_x(x) = p_z(f⁻¹(x)) · |det J_{f⁻¹}(x)|
Python Code
python 
import numpy as np
import torch

# ── Numerical Jacobian ────────────────────────────────────────────
def numerical_jacobian(f, x, h=1e-5):
    """Compute m×n Jacobian of f: ℝⁿ→ℝᵐ at x."""
    n  = len(x)
    f0 = f(x)
    m  = len(f0)
    J  = np.zeros((m, n))
    for j in range(n):
        e      = np.zeros(n); e[j] = h
        J[:, j] = (f(x + e) - f(x - e)) / (2*h)
    return J

def g(x):   # f: ℝ³ → ℝ²
    return np.array([x[0]**2 + x[1], x[1]*x[2] - x[0]])

x0 = np.array([1.0, 2.0, 3.0])
J  = numerical_jacobian(g, x0)
print("Jacobian:\n", J)

# ── Softmax Jacobian ──────────────────────────────────────────────
def softmax(z):
    e = np.exp(z - z.max())
    return e / e.sum()

def softmax_jacobian(z):
    s = softmax(z)
    return np.diag(s) - np.outer(s, s)   # ∂softmax/∂z

z_logits = np.array([1.0, 2.0, 0.5])
J_soft   = softmax_jacobian(z_logits)
print("\nSoftmax Jacobian:\n", J_soft.round(4))

# ── Torch jacobian (autograd) ─────────────────────────────────────
from torch.autograd.functional import jacobian

def torch_g(x): return torch.stack([x[0]**2+x[1], x[1]*x[2]-x[0]])
xt = torch.tensor([1.0, 2.0, 3.0])
J_torch = jacobian(torch_g, xt)
print("\nTorch Jacobian:\n", J_torch)

# ── Jacobian determinant (flow example) ──────────────────────────
# Measures volume change of a learned invertible transform
J_sq = np.array([[2.0, 0.5], [0.1, 3.0]])
log_det = np.log(abs(np.linalg.det(J_sq)))
print(f"\nlog|det J| = {log_det:.4f}")   # used in normalizing flows

07

Hessian Matrix Second-order curvature of the loss landscape

Theory

The Hessian H is the n×n matrix of all second-order partial derivatives. It encodes the curvature of the loss surface — how the gradient itself changes. Eigenvalues of H determine whether a critical point is a minimum (all positive), maximum (all negative), or saddle point (mixed signs).

Computing the full Hessian for a deep network (H ∈ ℝᵖˣᵖ, p = billions of params) is intractable, so ML uses Hessian-vector products and approximations (Fisher Information Matrix, Gauss-Newton).

Key Formulas
H_f(x) = [∂²f/∂xᵢ∂xⱼ] ∈ ℝⁿˣⁿ (symmetric by Clairaut's theorem)

-- Second-order Taylor approx (Newton step basis): f(x + Δx) ≈ f(x) + ∇f·Δx + ½ Δxᵀ H Δx

-- Newton's method step: Δx = -H⁻¹ ∇f

-- Condition number κ = λ_max/λ_min: High κ → ill-conditioned → slow gradient descent convergence
Python Code
python 
import numpy as np
import torch
from torch.autograd.functional import hessian

# ── Symbolic Hessian with SymPy ───────────────────────────────────
import sympy as sp
w1, w2 = sp.symbols('w1 w2')
f = w1**4 + w2**4 - 4*w1**2 - 4*w2**2 + w1*w2

H_sym = sp.hessian(f, [w1, w2])
print("Symbolic Hessian:\n", H_sym)

# ── Numerical Hessian ─────────────────────────────────────────────
def numerical_hessian(func, x, h=1e-4):
    n = len(x)
    H = np.zeros((n, n))
    for i in range(n):
        for j in range(n):
            ei, ej = np.zeros(n), np.zeros(n)
            ei[i] = h; ej[j] = h
            H[i, j] = (func(x+ei+ej) - func(x+ei-ej)
                      - func(x-ei+ej) + func(x-ei-ej)) / (4*h**2)
    return H

def loss_fn(x): return x[0]**4 + x[1]**4 - 4*x[0]**2 - 4*x[1]**2

x0 = np.array([1.5, 1.5])
H  = numerical_hessian(loss_fn, x0)
print("\nNumerical Hessian:\n", H)

eigvals = np.linalg.eigvals(H)
print(f"Eigenvalues: {eigvals}")
print(f"Condition number: {max(abs(eigvals))/min(abs(eigvals)):.2f}")

# ── Torch Hessian ─────────────────────────────────────────────────
def f_torch(x):
    return x[0]**4 + x[1]**4 - 4*x[0]**2 - 4*x[1]**2

xt = torch.tensor([1.5, 1.5], dtype=torch.float64)
H_torch = hessian(f_torch, xt)
print("\nTorch Hessian:\n", H_torch)

# ── Hessian-vector product (scalable) ────────────────────────────
# Avoids storing full H — O(n) instead of O(n²)
def hvp(f, x, v):
    """Hessian-vector product H·v using two backprop passes."""
    grad_f = torch.autograd.grad(f(x), x, create_graph=True)[0]
    return torch.autograd.grad(grad_f @ v, x)[0]

xp = torch.tensor([1.5, 1.5], dtype=torch.float64, requires_grad=True)
vp = torch.tensor([1.0, 0.0], dtype=torch.float64)
print("H·v =", hvp(f_torch, xp, vp))

08

Multivariate Taylor Series Local approximations of complex functions

Theory

Taylor series express a smooth function as an infinite polynomial around a point. The first-order approximation is a tangent plane; the second-order is a quadratic bowl. These approximations are the mathematical basis for gradient descent (1st order) and Newton's method (2nd order).

Key Formulas
-- Scalar Taylor expansion around x₀: f(x) = f(x₀) + ∇f(x₀)ᵀ(x-x₀) + ½(x-x₀)ᵀ H(x₀) (x-x₀) + O(||Δx||³)

-- 1st-order → gradient descent update: x_{t+1} = x_t - α ∇f(x_t)

-- 2nd-order → Newton update: x_{t+1} = x_t - H(x_t)⁻¹ ∇f(x_t)

-- Quadratic approximation of MSE loss (basis for Gauss-Newton): L(θ+δ) ≈ L(θ) + ∇Lᵀδ + ½ δᵀ(JᵀJ)δ
Python Code
python 
import numpy as np
import matplotlib.pyplot as plt

def f(x): return np.sin(x[0]) * np.cos(x[1])
def grad(x): return np.array([np.cos(x[0])*np.cos(x[1]),
                               -np.sin(x[0])*np.sin(x[1])])
def hess(x):
    return np.array([
        [-np.sin(x[0])*np.cos(x[1]), -np.cos(x[0])*np.sin(x[1])],
        [-np.cos(x[0])*np.sin(x[1]),  -np.sin(x[0])*np.cos(x[1])]
    ])

x0 = np.array([1.0, 0.5])

# 1st and 2nd order Taylor approximations
def taylor_1st(x):
    d = x - x0
    return f(x0) + grad(x0) @ d

def taylor_2nd(x):
    d = x - x0
    return f(x0) + grad(x0) @ d + 0.5 * d @ hess(x0) @ d

# Evaluate along a line from x0

Gradient Descent & Variants Iterative optimization via gradient steps

Theory

Gradient descent iteratively moves parameters opposite to the gradient to minimize loss. Vanilla GD uses the full dataset; SGD uses one or mini-batch samples. Momentum accumulates gradient history to escape local curvature. RMSProp adapts learning rates per parameter. Adam combines momentum and RMSProp — the workhorse of deep learning.

Key Formulas
-- Gradient Descent: θ_{t+1} = θ_t - α ∇L(θ_t)

-- Momentum (Polyak): v_{t+1} = β v_t + ∇L(θ_t) θ_{t+1} = θ_t - α v_{t+1}

-- Adam (Kingma & Ba 2015): m_t = β₁ m_{t-1} + (1-β₁)∇L (1st moment) v_t = β₂ v_{t-1} + (1-β₂)(∇L)² (2nd moment) m̂_t = m_t/(1-β₁ᵗ) (bias correction) v̂_t = v_t/(1-β₂ᵗ) θ_t = θ_{t-1} - α m̂_t / (√v̂_t + ε)

import numpy as np
import matplotlib.pyplot as plt

# ── Implement GD, Momentum, Adam from scratch ────────────────────
def rosenbrock(x):    # classic non-convex test function
    return (1 - x[0])**2 + 100*(x[1] - x[0]**2)**2

def rosenbrock_grad(x):
    g0 = -2*(1-x[0]) - 400*x[0]*(x[1]-x[0]**2)
    g1 = 200*(x[1] - x[0]**2)
    return np.array([g0, g1])

x_start = np.array([-1.0, 1.0])
n_steps = 2000

# Plain GD
def run_gd(alpha=1e-3):
    x = x_start.copy(); path = [x.copy()]
    for _ in range(n_steps):
        x -= alpha * rosenbrock_grad(x); path.append(x.copy())
    return np.array(path)

# Adam
def run_adam(alpha=0.01, b1=0.9, b2=0.999, eps=1e-8):
    x = x_start.copy()
    m, v, path = np.zeros(2), np.zeros(2), [x.copy()]
    for t in range(1, n_steps+1):
        g    = rosenbrock_grad(x)
        m    = b1*m + (1-b1)*g
        v    = b2*v + (1-b2)*g**2
        m_hat = m / (1 - b1**t)
        v_hat = v / (1 - b2**t)
        x    -= alpha * m_hat / (np.sqrt(v_hat) + eps)
        path.append(x.copy())
    return np.array(path)

Lagrange Multipliers Constrained optimization in parameter space

Theory

Lagrange multipliers find the extrema of f(x) subject to equality constraints g(x) = 0. At the optimum, the gradient of f is parallel to the gradient of g — i.e., ∇f = λ∇g. The Lagrangian L = f - λg combines objective and constraint.

For inequality constraints (g(x) ≤ 0), the KKT (Karush-Kuhn-Tucker) conditions generalize this — they underpin SVMs, constrained RL, and safety-constrained neural networks.

Key Formulas
-- Lagrangian (equality): ℒ(x, λ) = f(x) - λ g(x) ∇_x ℒ = 0 → ∇f = λ ∇g ∂ℒ/∂λ = 0 → g(x) = 0

-- KKT conditions (inequality, g(x) ≤ 0): ∇f(x*) + μ ∇g(x*) = 0 μ ≥ 0, g(x*) ≤ 0, μ g(x*) = 0 (complementary slackness)

-- SVM dual (max-margin classifier): max_α Σαᵢ - ½ ΣΣ αᵢαⱼyᵢyⱼ xᵢᵀxⱼ s.t. 0 ≤ αᵢ ≤ C, Σ αᵢyᵢ = 0

Divergence & Curl Vector field analysis and flow dynamics

Theory

Divergence (∇·F) measures local expansion or contraction of a vector field — a positive divergence means the field is "spreading out" (source), negative means "converging" (sink). In ML, the gradient field of the loss has divergence related to the Laplacian of the loss.

Curl (∇×F) measures rotation in a vector field. Gradient fields are irrotational (zero curl) — a fundamental property exploited in energy-based models. Curl appears in physics-informed ML for fluid dynamics simulation.

Key Formulas
-- Divergence (ℝ³): ∇·F = ∂F_x/∂x + ∂F_y/∂y + ∂F_z/∂z

-- Laplacian (divergence of gradient): ∇²f = Σᵢ ∂²f/∂xᵢ² = tr(H)

-- Curl (ℝ³): ∇×F = (∂F_z/∂y - ∂F_y/∂z, ∂F_x/∂z - ∂F_z/∂x, ∂F_y/∂x - ∂F_x/∂y)

-- Instantaneous change of log-density (Neural ODE flows): d log p/dt = -∇·f_θ (Liouville equation)

Backpropagation via Chain Rule The algorithm that trains every deep learning model

Theory

Backpropagation is the efficient application of the multivariate chain rule to compute gradients of the loss w.r.t. all parameters in a neural network in a single backward pass. Forward pass computes activations; backward pass propagates gradient signals layer by layer. Modern frameworks (PyTorch, JAX) build computation graphs during the forward pass and traverse them in reverse during backward.

The key insight: reverse-mode autodiff computes the Jacobian-vector product Jᵀv at cost O(1) regardless of output dimension — perfect for scalar losses in ML.

Key Formulas
-- Layer notation: a⁽ˡ⁾ = σ(W⁽ˡ⁾ a⁽ˡ⁻¹⁾ + b⁽ˡ⁾)

-- Output layer delta: δ⁽ᴸ⁾ = ∇_a L ⊙ σ'(z⁽ᴸ⁾)

-- Hidden layer delta (backward pass): δ⁽ˡ⁾ = ((W⁽ˡ⁺¹⁾)ᵀ δ⁽ˡ⁺¹⁾) ⊙ σ'(z⁽ˡ⁾)

-- Weight gradient: ∂L/∂W⁽ˡ⁾ = δ⁽ˡ⁾ (a⁽ˡ⁻¹⁾)ᵀ ∂L/∂b⁽ˡ⁾ = δ⁽ˡ⁾
Python Code
python 
import numpy as np

# ── Full forward + backward for 3-layer network ───────────────────
class NeuralNet:
    def __init__(self, layer_sizes):
        np.random.seed(42)
        self.params = {}
        for l in range(1, len(layer_sizes)):
            n_in  = layer_sizes[l-1]
            n_out = layer_sizes[l]
            self.params[f'W{l}'] = np.random.randn(n_out, n_in) * 0.1
            self.params[f'b{l}'] = np.zeros((n_out, 1))

    def sigmoid(self, z): return 1/(1+np.exp(-z))

    def forward(self, X):
        self.cache = {'a0': X}
        a = X
        L = len([k for k in self.params if k.startswith('W')])
        for l in range(1, L+1):
            z = self.params[f'W{l}'] @ a + self.params[f'b{l}']
            a = self.sigmoid(z)
            self.cache[f'z{l}'] = z
            self.cache[f'a{l}'] = a
        return a

    def backward(self, X, y):
        m = X.shape[1]
        L = len([k for k in self.params if k.startswith('W')])
        grads = {}

        # Output layer delta
        a_L   = self.cache[f'a{L}']
        sig_p = a_L * (1 - a_L)               # σ'(z)
        delta = (a_L - y) * sig_p               # δ⁽ᴸ⁾

        for l in reversed(range(1, L+1)):
            a_prev            = self.cache[f'a{l-1}']
            grads[f'dW{l}']  = (1/m) * delta @ a_prev.T
            grads[f'db{l}']  = (1/m) * delta.sum(axis=1, keepdims=True)
            if l > 1:
                a_prev_l   = self.cache[f'a{l-1}']
                sig_p_prev = a_prev_l * (1 - a_prev_l)
                delta      = (self.params[f'W{l}'].T @ delta) * sig_p_prev
        return grads

KL Divergence & Information Geometry Calculus in the space of probability distributions

Theory

KL divergence D_KL(P||Q) measures the "distance" from distribution Q to P (asymmetric). Its gradient w.r.t. parameters defines the natural gradient direction — steepest ascent in distribution space rather than parameter space. The Fisher Information Matrix is the Hessian of KL divergence at P=Q, defining the Riemannian metric of the statistical manifold.

Information geometry treats the space of probability distributions as a curved manifold — with geodesics, curvature, and parallel transport — all powered by multivariate calculus.

Key Formulas
-- KL Divergence: D_KL(P||Q) = Σ P(x) log(P(x)/Q(x)) (discrete) = ∫ p(x) log(p(x)/q(x)) dx (continuous)

-- Fisher Information Matrix: F(θ) = E_x[∇_θ log p(x|θ) ∇_θ log p(x|θ)ᵀ] = -E_x[∂²log p(x|θ)/∂θᵢ∂θⱼ]

-- Natural gradient step: θ_{t+1} = θ_t - α F(θ)⁻¹ ∇L(θ)

-- ELBO (VAE objective): ELBO = E_q[log p(x|z)] - D_KL(q(z|x) || p(z))

import numpy as np
import torch
import torch.nn.functional as F

# ── KL Divergence (discrete) ──────────────────────────────────────
def kl_divergence(P, Q, eps=1e-10):
    P, Q = np.array(P)+eps, np.array(Q)+eps
    P /= P.sum(); Q /= Q.sum()
    return np.sum(P * np.log(P/Q))

P = [0.4, 0.35, 0.25]
Q = [0.33, 0.33, 0.34]
print(f"KL(P||Q) = {kl_divergence(P, Q):.4f}")

def kl_gaussian(mu1, sigma1, mu2, sigma2):
    """KL(N(μ₁,σ₁²) || N(μ₂,σ₂²))"""
    return (np.log(sigma2/sigma1)
            + (sigma1**2 + (mu1-mu2)**2)/(2*sigma2**2)
            - 0.5)

# ── VAE reparameterization + ELBO ────────────────────────────────
def vae_elbo(x, mu_z, log_var_z, x_recon):
    """
    ELBO = Reconstruction loss + KL divergence
    KL(N(μ,σ²) || N(0,1)) = -½ Σ(1 + log σ² - μ² - σ²)
    """
    recon_loss = F.mse_loss(x_recon, x, reduction='sum')
    kl_loss    = -0.5 * torch.sum(1 + log_var_z - mu_z**2 - log_var_z.exp())
    return recon_loss + kl_loss

Convexity & Saddle Points Geometry of loss landscapes and critical points

Theory

A function f is convex if the line segment between any two points on its graph lies above the graph: f(λx + (1-λ)y) ≤ λf(x) + (1-λ)f(y). Convex functions have a single global minimum — gradient descent is guaranteed to converge. Most deep learning losses are non-convex, riddled with saddle points and local minima.

A saddle point has ∇f = 0 but the Hessian has both positive and negative eigenvalues. In high dimensions, saddle points are exponentially more common than local minima. Fortunately, they are often escapable with noise (SGD) or second-order curvature information.

Strict saddle property: many non-convex ML problems (matrix factorization, shallow nets) satisfy this — all local minima are global, and saddle points have a direction of negative curvature enabling escape.

Key Formulas
-- Convexity condition (twice differentiable): f convex ⟺ H(x) ⪰ 0 (positive semi-definite) ∀x

-- Jensen's inequality: f(E[X]) ≤ E[f(X)] (for convex f)

-- Critical point classification via Hessian: All λᵢ(H) > 0 → local minimum All λᵢ(H) < 0 → local maximum Mixed signs → saddle point

-- Strong convexity (guarantees linear convergence): f(y) ≥ f(x) + ∇f(x)ᵀ(y-x) + (μ/2)||y-x||², μ > 0

# ── Check convexity via Hessian eigenvalues ───────────────────────
def is_convex_at(hessian_fn, points):
    """Check PSD of Hessian at multiple points."""
    results = []
    for x in points:
        H       = hessian_fn(x)
        eigvals = np.linalg.eigvals(H)
        results.append({'x': x, 'eigvals': eigvals,
                         'convex': bool(np.all(eigvals >= -1e-8))})

Implicit Differentiation & Implicit Function Theorem Differentiating through constraints and equilibrium conditions

Theory

The Implicit Function Theorem (IFT) guarantees that if F(x, y) = 0 defines y implicitly as a function of x near a point, then dy/dx = -∂F/∂x / ∂F/∂y. This allows differentiation without explicitly solving for y — critical when the relationship between variables is defined by an optimization problem or fixed-point equation.

In ML, implicit differentiation through optimization problems underpins bi-level optimization, meta-learning, hyperparameter optimization, and Neural ODEs where the "layer" is defined as the solution to a differential equation.

Key Formulas
-- IFT: F(x, y(x)) = 0 ⟹ dy/dx = -(∂F/∂x) / (∂F/∂y)

-- Multivariate IFT: ∂y/∂x = -(∂F/∂y)⁻¹ (∂F/∂x)

-- Bi-level optimization (meta-learning / hyperparam opt): min_φ L_val(θ*(φ), φ) s.t. θ*(φ) = argmin_θ L_train(θ, φ)

-- Gradient via IFT (avoiding unrolling): dL_val/dφ = ∂L_val/∂φ - (∂L_val/∂θ)(∂²L_train/∂θ²)⁻¹(∂²L_train/∂θ∂φ)

import torch

# F(x, y) = x² + y² - r² = 0

print(f"Explicit dy/dx: {dy_dx_explicit:.6f}")

# ── Differentiating through an inner optimization (IFT approach) ──
def inner_opt(phi, n_steps=100, lr=0.1):
    """Inner optimization: θ*(φ) = argmin_θ (θ - φ)² + θ⁴"""
    theta = np.array([0.0])
    for _ in range(n_steps):
        grad  = 2*(theta - phi) + 4*theta**3
        theta = theta - lr * grad
    return theta

def val_loss(theta, phi):
    """Validation loss."""
    return (theta - (phi + 1))**2

d2L_dtheta_dphi = -2

Change of Variables & Probability Transformations Transforming distributions through learned mappings

Theory

When a random variable X is transformed via a differentiable function g to Y = g(X), the probability density of Y is related to that of X via the change-of-variables formula, which involves the Jacobian determinant of g⁻¹. This is the mathematical heart of normalizing flows — a class of deep generative models that learn complex distributions by composing simple, invertible transformations.

The absolute value of the Jacobian determinant measures how much the transformation stretches or compresses volume in probability space — preserving the total probability mass.

Key Formulas
-- Change of variables (1D): p_Y(y) = p_X(g⁻¹(y)) · |d/dy g⁻¹(y)|

-- Multivariate change of variables: p_Y(y) = p_X(g⁻¹(y)) · |det J_{g⁻¹}(y)|

-- Log-likelihood (normalizing flow): log p_X(x) = log p_Z(f(x)) + log|det J_f(x)|

-- Composition of K flows: log p(x) = log p_Z(z_K) + Σₖ log|det J_{fₖ}(zₖ)|

-- Reparameterization trick (VAE): z = μ + σ ⊙ ε, ε ~ N(0, I) ← differentiable sampling

class PlanarFlow(nn.Module):
    def __init__(self, dim):
        super().__init__()
        self.w = nn.Parameter(torch.randn(dim))
        self.u = nn.Parameter(torch.randn(dim))
        self.b = nn.Parameter(torch.zeros(1))

    def forward(self, z):
        """f(z) = z + u·tanh(wᵀz + b)"""
        lin      = z @ self.w + self.b          # (batch,)
        z_new    = z + self.u * torch.tanh(lin).unsqueeze(1)

        psi      = (1 - torch.tanh(lin)**2).unsqueeze(1) * self.w
        log_det  = torch.log((1 + psi @ self.u.unsqueeze(1)).abs() + 1e-8)
        return z_new, log_det.squeeze()

Matrix Calculus Derivatives of matrix and tensor expressions

Theory

Matrix calculus extends differentiation to functions involving matrices and vectors. In deep learning, every layer involves matrix operations, and we need derivatives like ∂(Xw)/∂w, ∂(tr(AB))/∂A, ∂||Ax-b||²/∂A efficiently. The trace trick — tr(aᵀb) = aᵀb for vectors — and identities for derivatives of determinants and inverses appear repeatedly in probabilistic ML (Gaussian log-likelihoods, covariance matrix learning).

Key Formulas
-- Vector-by-vector (Jacobian): ∂(Ax)/∂x = Aᵀ (numerator layout) ∂(xᵀA)/∂x = A

-- Matrix-by-scalar differentials: ∂ log|det A|/∂A = (Aᵀ)⁻¹ = (A⁻¹)ᵀ ∂ tr(AB)/∂A = Bᵀ ∂ tr(AᵀB)/∂A = B

Automatic Differentiation (Autograd) Exact gradients at machine precision via computation graphs

Theory

Automatic differentiation (AD) is neither symbolic differentiation (SymPy) nor finite differences — it applies the chain rule to elementary operations in exact arithmetic. In forward mode, a tangent value propagates alongside each primal value (efficient for few inputs). In reverse mode (backprop), an adjoint propagates backward (efficient for few outputs, like a scalar loss).

Modern frameworks (PyTorch, JAX, TensorFlow) implement reverse-mode AD via dynamic or static computation graphs. JAX additionally supports forward-mode via jvp, higher-order differentiation, and just-in-time compilation via XLA — enabling Hessian-vector products, per-sample gradients, and functional transformations.

Key Formulas
-- Forward mode (JVP — Jacobian-vector product): ẋ → ḟ = J_f · ẋ (one pass per input direction)

-- Reverse mode (VJP — vector-Jacobian product): v̄ → x̄ = J_f^T · v̄ (one pass per output)

-- Hessian-vector product (efficient 2nd order): Hv = ∂/∂ε [∇f(x + εv)]|_{ε=0} (forward-over-reverse)

-- Per-sample gradients (privacy, influence): {∂L(xᵢ,yᵢ)/∂θ}_{i=1}^m (vmap over batch)

Stochastic Calculus & Langevin Dynamics Randomness, SDEs, and diffusion in optimization

Theory

Stochastic Differential Equations (SDEs) add a noise term to ordinary differential equations: dx = f(x,t)dt + g(x,t)dW, where W is a Wiener process (Brownian motion). Itô's lemma is the stochastic chain rule — it adds a correction term proportional to the second derivative due to the quadratic variation of Brownian motion (dW² = dt).

Langevin dynamics — gradient descent with Gaussian noise — samples from the posterior of parameters, bridging optimization and Bayesian inference. This is the foundation of score-based diffusion models (DDPM, DDIM, Stable Diffusion).

Key Formulas
-- Stochastic Gradient Descent (SDE perspective): dθ = -∇L(θ) dt + √(2T) dW (Langevin diffusion)

-- Itô's Lemma (stochastic chain rule): df(X_t) = (∂f/∂t + μ ∂f/∂x + ½σ² ∂²f/∂x²) dt + σ ∂f/∂x dW_t

-- Score-based diffusion (forward SDE): dx = f(x,t) dt + g(t) dW (add noise)

-- Reverse SDE (denoising / generation): dx = [f(x,t) - g(t)² ∇_x log p_t(x)] dt + g(t) dW̄

-- Score function (trained neural network): s_θ(x, t) ≈ ∇_x log p_t(x)

import matplotlib.pyplot as plt

def U(theta):  # Potential = -log p(θ)
    return 0.5*(theta[0]**2) + 2*(theta[1]-1)**2

def grad_U(theta):
    return np.array([theta[0], 4*(theta[1]-1)])

print(f"Sample std:   {samples.std(axis=0).round(3)}")    # ≈ [√2, ½]

    x_t     = alpha_t * x0 + np.sqrt(1 - alpha_t**2) * noise

Variational Calculus & Functional Derivatives Optimizing over functions, not just parameters

Theory

Calculus of variations finds functions that extremize functionals (functions of functions). The Euler-Lagrange equation gives the necessary condition. In ML, this appears when we optimize over distributions (variational inference), policies (RL), or density functions directly.

The functional derivative δF/δf(x) is the generalization of the partial derivative to continuous function spaces — it measures how a functional changes when f is perturbed by a delta function at x. This is central to Gaussian Process inference, optimal transport, and infinite-width neural network theory.

Key Formulas
-- Functional: F[f] = ∫ L(x, f(x), f'(x)) dx

-- Euler-Lagrange equation: ∂L/∂f - d/dx(∂L/∂f') = 0

-- Functional derivative: δF/δf(x) = ∂L/∂f(x) - d/dx(∂L/∂f'(x))

-- Entropy functional: H[p] = -∫ p(x) log p(x) dx δH/δp(x) = -(log p(x) + 1)

-- Optimal policy (entropy-regularised RL): π*(a|s) ∝ exp(Q(s,a) / α) (Boltzmann distribution)

import numpy as np
from scipy.optimize import minimize
import torch

# ── Variational Inference: minimize KL via functional derivative ───
# Find q* = argmin KL(q||p) in parametric family q ~ N(μ, σ²)
# → Exact posterior matching for Gaussian likelihood

def elbo_gaussian(params, x_obs, prior_mu=0, prior_sigma=1, likelihood_sigma=0.5):
    """ELBO for Gaussian model via closed-form functional derivative."""
    mu, log_sigma = params[0], params[1]
    sigma = np.exp(log_sigma)

    # Expected log-likelihood: E_q[log p(x|θ)]
    recon = -0.5 * np.mean((x_obs - mu)**2) / likelihood_sigma**2

    # KL divergence: KL(N(μ,σ²) || N(0,1))
    kl    = -0.5 * (1 + 2*log_sigma - mu**2 - sigma**2)

    return -(recon - kl)   # negate to minimize

np.random.seed(0)
x_data  = np.random.randn(50) * 0.5 + 2.0   # true μ = 2
result  = minimize(elbo_gaussian, [0.0, 0.0], args=(x_data,), method='L-BFGS-B')
mu_vi, sigma_vi = result.x[0], np.exp(result.x[1])
print(f"VI posterior: μ={mu_vi:.3f}, σ={sigma_vi:.3f}")
print(f"Data mean:    {x_data.mean():.3f}")

# ── Entropy-regularised RL: soft-max optimal policy ───────────────
Q_values = np.array([1.5, 2.3, 0.8, 3.1])   # Q(s, a) for 4 actions
alpha    = 1.0                                  # temperature

# Optimal policy: π* ∝ exp(Q/α)  (Boltzmann / softmax)
pi_star  = np.exp(Q_values / alpha)
pi_star /= pi_star.sum()
print(f"\nOptimal policy (α={alpha}): {pi_star.round(3)}")
print(f"Entropy H[π*] = {-np.sum(pi_star * np.log(pi_star)):.4f}")

# ── Maximum entropy principle: find p* maximizing H subject to constraints
from scipy.optimize import minimize

def neg_entropy(p): return np.sum(p * np.log(p + 1e-10))

# Max entropy distribution over 5 outcomes with mean constraint E[X]=2
x_vals = np.array([0.0, 1.0, 2.0, 3.0, 4.0])
constraints = [
    {'type': 'eq', 'fun': lambda p: p.sum() - 1},
    {'type': 'eq', 'fun': lambda p: p @ x_vals - 2}
]
bounds = [(1e-8, 1)] * 5
p_maxent = minimize(neg_entropy, np.ones(5)/5,
                   constraints=constraints, bounds=bounds).x
print(f"\nMax-entropy p* (mean=2): {p_maxent.round(3)}")
print(f"Exponential family form:  {np.exp(-p_maxent).round(3)}")

Integral Transforms: Fourier & Laplace Frequency-domain analysis for signals and attention

Theory

The Fourier Transform decomposes a function into its frequency components: F̂(ω) = ∫ f(x) e^{-2πiωx} dx. Convolution in the spatial domain becomes multiplication in the frequency domain — making CNNs analyzable as frequency filters. The Laplace Transform generalizes Fourier to complex frequencies and is used in stability analysis of dynamical systems and neural ODEs.

In the context of attention mechanisms, Fourier-based positional encodings (sinusoidal embeddings in Transformers, RoPE) and spectral graph convolutions (Graph Neural Networks) are direct applications. Random Fourier Features approximate kernel functions using Bochner's theorem, enabling scalable kernel machines.

Key Formulas
-- Fourier Transform: F̂(ω) = ∫_{-∞}^{∞} f(t) e^{-2πiωt} dt f(t) = ∫_{-∞}^{∞} F̂(ω) e^{2πiωt} dω

-- Convolution theorem: (f * g)(t) ↔ F̂(ω) · Ĝ(ω)

-- Sinusoidal positional encoding (Transformer): PE(pos, 2i) = sin(pos / 10000^{2i/d}) PE(pos, 2i+1) = cos(pos / 10000^{2i/d})

-- RoPE (Rotary Position Embedding): f(x, m) = x · e^{imθ} (complex rotation by position m)

-- Random Fourier Features (kernel approximation): k(x,y) ≈ φ(x)ᵀφ(y), φ(x) = √(2/D) cos(Ωᵀx + b)

import numpy as np
import torch
import matplotlib.pyplot as plt

# ── Sinusoidal Positional Encoding (Transformer) ──────────────────
def sinusoidal_pe(seq_len, d_model):
    PE  = np.zeros((seq_len, d_model))
    pos = np.arange(seq_len)[:, None]
    i   = np.arange(0, d_model, 2)[None, :]
    div = np.power(10000, i / d_model)
    PE[:, 0::2] = np.sin(pos / div)
    PE[:, 1::2] = np.cos(pos / div)
    return PE

PE = sinusoidal_pe(100, 64)
print(f"PE shape: {PE.shape}")
print(f"PE[0, :4] = {PE[0, :4].round(3)}")

# ── RoPE (Rotary Positional Encoding) ────────────────────────────
def rope_rotate(x, pos, base=10000):
    """Apply RoPE to query/key vector x at position pos."""
    d  = x.shape[-1]
    i  = torch.arange(0, d//2, dtype=torch.float32)
    theta = 1.0 / (base ** (2*i / d))
    angle = pos * theta
    cos_a = torch.cos(angle)
    sin_a = torch.sin(angle)
    x1, x2 = x[..., :d//2], x[..., d//2:]
    return torch.cat([x1*cos_a - x2*sin_a,
                       x1*sin_a + x2*cos_a], dim=-1)

q  = torch.randn(8)   # query vector
q_rope = rope_rotate(q, pos=5)
print(f"\nOriginal q:     {q[:4].numpy().round(3)}")
print(f"RoPE q (pos=5): {q_rope[:4].detach().numpy().round(3)}")

# ── Random Fourier Features (kernel approximation) ────────────────
def random_fourier_features(X, D=1000, sigma=1.0):
    """
    Approximate RBF kernel: k(x,y) ≈ φ(x)ᵀφ(y)
    where φ(x) = √(2/D) [cos(ω₁ᵀx+b₁), ..., cos(ωDᵀx+bD)]
    """
    d    = X.shape[1]
    Omega = np.random.randn(d, D) / sigma
    b     = np.random.uniform(0, 2*np.pi, D)
    Phi   = np.sqrt(2.0/D) * np.cos(X @ Omega + b)
    return Phi

np.random.seed(1)
X_test = np.random.randn(100, 5)
Phi    = random_fourier_features(X_test, D=2000)

# Exact RBF kernel vs approximation
x1, x2 = X_test[0], X_test[1]
k_exact = np.exp(-np.linalg.norm(x1-x2)**2 / 2)
k_approx = Phi[0] @ Phi[1]
print(f"\nExact RBF kernel:  {k_exact:.4f}")
print(f"RFF approximation: {k_approx:.4f}")

# ── FFT for CNN filter analysis ───────────────────────────────────
kernel    = np.array([1, -2, 1], dtype=float)   # Laplacian (edge detector)
K_freq    = np.abs(np.fft.fft(kernel, n=64))
freqs     = np.fft.fftfreq(64)
print(f"\nLaplacian kernel peak frequency: {freqs[np.argmax(K_freq)]:.2f}")
print("→ High-pass filter (edge detection), as expected")
