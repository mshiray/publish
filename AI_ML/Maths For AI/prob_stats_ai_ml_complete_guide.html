Probability Fundamentals

The mathematical language for expressing uncertainty — essential for every probabilistic model in AI.

Theory

What is Probability?

Probability is a measure on a sample space Ω — the set of all possible outcomes. An event A is a subset of Ω, and P(A) ∈ [0,1] quantifies the likelihood of that event. Kolmogorov's three axioms make this rigorous: non-negativity, unit measure on Ω, and countable additivity for disjoint events.

Conditional probability P(A|B) = P(A∩B)/P(B) updates belief about A given that B has occurred. This is the heart of Bayesian reasoning. Two events are independent iff P(A∩B) = P(A)·P(B) — knowing B tells you nothing about A.

Bayes' Theorem P(H|E) = P(E|H)·P(H)/P(E) is the fundamental rule for rational belief update. P(H) is the prior, P(E|H) the likelihood, P(E) the normalizing evidence, and P(H|E) the posterior.

  • The chain rule P(A₁∩…∩Aₙ) = P(A₁)·P(A₂|A₁)·…·P(Aₙ|A₁,…,Aₙ₋₁) is how language models factorize sequence probability
  • The law of total probability P(B) = Σᵢ P(B|Aᵢ)P(Aᵢ) marginalizes over a partition {Aᵢ}
  • Independence vs. conditional independence: X⊥Y|Z means X and Y are independent given Z — the key assumption in Naive Bayes and graphical models

import numpy as np

rng   = np.random.default_rng(42)
rolls = rng.integers(1, 7, size=100_000)

P_even = np.mean(rolls % 2 == 0)         # ~0.5
P_gt3  = np.mean(rolls > 3)               # ~0.5
P_both = np.mean((rolls % 2 == 0) & (rolls > 3))

# Union rule: P(A∪B) = P(A) + P(B) − P(A∩B)
P_union = P_even + P_gt3 - P_both
print(f"P(even)={P_even:.3f}  P(>3)={P_gt3:.3f}")
print(f"P(even ∪ >3) = {P_union:.3f}")
print(f"P(complement of even) = {1-P_even:.3f}")

Posterior Inference — Medical Test Example

Low base-rate means even a highly accurate test has surprising false-positive rates. The same math drives LLM token scoring.

Bayes' Theorem
P(H|E) = P(E|H) · P(H) / [P(E|H)·P(H) + P(E|¬H)·P(¬H)] 
# P(spam)=0.4, P("free"|spam)=0.7, P("free"|ham)=0.1
P_spam            = 0.4
P_free_spam       = 0.7
P_free_ham        = 0.1
P_ham             = 1 - P_spam

# Total probability of observing "free"
P_free  = P_free_spam * P_spam + P_free_ham * P_ham

# Posterior — classic Naive Bayes step
P_spam_given_free = (P_free_spam * P_spam) / P_free
print(f"P(spam | 'free') = {P_spam_given_free:.4f}")  # 0.8235

# Log-likelihood (numerically stable, used in production)
import math
log_odds = (math.log(P_free_spam/P_free_ham) +
            math.log(P_spam/P_ham))
print(f"Log-odds: {log_odds:.4f}")  # +ve → lean spam

# Chain rule: P(w1,w2,w3) = P(w1)·P(w2|w1)·P(w3|w1,w2)
# This is how GPT scores a sentence!
log_P_sentence = math.log(0.05) + math.log(0.3) + math.log(0.4)
print(f"Log P(sentence) = {log_P_sentence:.4f}")

Probability Distributions

Parametric families that describe the randomness in data, outputs, and latent variables throughout all of ML.

Theory

Why Distributions Are the Core Language of ML

Every ML model is implicitly or explicitly a probability distribution. Supervised models estimate P(y|x). Generative models learn P(x). Reinforcement learning agents model P(reward|state, action). Understanding which distribution governs a variable dictates your loss function, link function, sampling strategy, and regularizer.

A probability mass function (PMF) gives point probabilities for discrete variables: P(X=k). A probability density function (PDF) f(x) gives density for continuous variables — probabilities are integrals: P(a≤X≤b) = ∫f(x)dx. The CDF F(x)=P(X≤x) is non-decreasing, right-continuous, from 0 to 1. Its inverse is the quantile/PPF function used in sampling.

Key moment relationships: Mean E[X] = first moment. Variance Var(X) = E[X²]−E[X]² = second central moment. A distribution is fully characterized by its moment-generating function or characteristic function. The Normal distribution is unique as the MGF exponential family member with maximum entropy given fixed mean and variance.

  • Exponential family: Normal, Bernoulli, Poisson, Gamma, Beta, Dirichlet — all share the form p(x|η) = h(x)·exp(η·T(x)−A(η)). This structure enables GLMs and variational inference.
  • Location-scale families: Normal(μ,σ²), Cauchy, Laplace — if X~F(0,1), then μ+σX follows the location-scale version.
  • Conjugate pairs: Beta–Binomial, Dirichlet–Categorical, Normal–Normal, Gamma–Poisson — posterior has the same form as prior; critical for Bayesian models.

logits = np.array([2.1, 1.4, 0.3, -0.5, 1.9])

def softmax_sample(logits, temperature=1.0):
    scaled  = logits / temperature
    exp_l   = np.exp(scaled - scaled.max())  # numerical stability
    probs   = exp_l / exp_l.sum()
    return np.random.choice(len(probs), p=probs), probs

for T in [0.1, 0.7, 1.0, 2.0]:
    idx, probs = softmax_sample(logits, T)
    entropy = -np.sum(probs * np.log(probs + 1e-9))

# Low T → peaked distribution (greedy), high T → uniform (random)

Descriptive Statistics

Summarizing data distributions — the critical first step in any EDA, feature engineering, or model audit pipeline.

Theory

Moments, Shape, and Correlation

Descriptive statistics condense a dataset into a small number of interpretable quantities. The first four standardized moments characterize a distribution: mean (location), variance (spread), skewness (asymmetry), and kurtosis (tail heaviness).

Variance Var(X) = E[(X−μ)²]. The sample estimator uses n−1 (Bessel's correction) to be unbiased. Standard deviation σ is in the same units as X. The coefficient of variation σ/μ is scale-free — useful when comparing features with different magnitudes.

Skewness = E[(X−μ)³]/σ³. Positive skew means a right tail (income distributions, latency). Negative skew means a left tail. Most tree-based ML models are robust to skew, but linear/logistic regression benefits from log or Box-Cox transformation.

Kurtosis = E[(X−μ)⁴]/σ⁴ − 3 (excess). High kurtosis (leptokurtic) means heavy tails — financial returns, NLP attention scores. Low kurtosis (platykurtic) means thin tails.

Pearson correlation r = Cov(X,Y)/(σₓσᵧ) measures linear association. Spearman ρ uses rank correlations — robust to outliers and non-linearity. Kendall τ counts concordant vs discordant pairs — better for small samples.

  • The covariance matrix Σ captures all pairwise linear relationships. PCA diagonalizes it via eigendecomposition.
  • Robust statistics: median, MAD (median absolute deviation), trimmed mean — critical when data has outliers that would bias neural network training.
  • Quantiles divide data into equal-probability buckets. Quantile normalization is a standard preprocessing step in genomics and NLP embedding analysis.

# ── LayerNorm (as used in Transformers) ──
def layer_norm(x, eps=1e-5):
    """Normalize along last dimension — used in every Transformer."""
    mean = x.mean(axis=-1, keepdims=True)
    var  = x.var(axis=-1, keepdims=True)
    return (x - mean) / np.sqrt(var + eps)

activations = np.random.randn(8, 512)  # batch=8, d_model=512

print(f"After LN: mean≈{normed.mean():.6f}  std≈{normed.std():.6f}")

Statistical Inference

Drawing conclusions about populations from samples — foundation of model evaluation, A/B testing, and experiment design.

Theory

Estimation, CLT, and the Bootstrap

Statistical inference aims to learn population parameters θ from a finite sample. Point estimation gives a single best guess θ̂. Interval estimation gives a range [L, U] that contains θ with a specified probability (confidence level).

Maximum Likelihood Estimation (MLE): θ̂_MLE = argmax_θ Σ log p(xᵢ|θ). It finds parameters that make the observed data most probable. MLE is the backbone of neural network training — cross-entropy loss IS negative log-likelihood.

The Central Limit Theorem states that for i.i.d. samples from ANY distribution with finite mean μ and variance σ², the sample mean X̄ satisfies: √n(X̄−μ)/σ → Normal(0,1) as n→∞. This justifies t-tests and z-tests for large-enough n regardless of the population shape.

The Bootstrap is a distribution-free alternative: resample with replacement B times (B=10,000), compute θ̂ on each resample, and use the empirical percentiles as the confidence interval. No distributional assumptions needed.

  • Bias-variance tradeoff: MSE(θ̂) = Bias²(θ̂) + Var(θ̂). MLE is asymptotically unbiased but may have high variance for small n. Regularized estimators (MAP, ridge) introduce bias to reduce variance.
  • Consistency: an estimator θ̂ₙ → θ in probability. MLE is consistent under mild regularity conditions. SGD in neural nets exploits this.
  • Fisher information I(θ) = −E[∂²log p/∂θ²] bounds the variance of any unbiased estimator (Cramér-Rao bound). Appears in natural gradient descent (K-FAC, Shampoo optimizers).

MLE + CI

Point Estimation & Confidence Intervals

from scipy import stats
import numpy as np

data = np.random.normal(5, 2, 500)

# MLE for Gaussian
mu_mle    = data.mean()
sigma_mle = data.std(ddof=1)           # unbiased

# 95% t-based CI for mean
se   = stats.sem(data)
ci95 = stats.t.interval(0.95, df=len(data)-1,
                          loc=mu_mle, scale=se)
print(f"μ̂={mu_mle:.3f}  σ̂={sigma_mle:.3f}")
print(f"95% CI: [{ci95[0]:.3f}, {ci95[1]:.3f}]")

# Bootstrap CI — no distributional assumption
rng = np.random.default_rng(42)
boot_means = [rng.choice(data, len(data)).mean()
              for _ in range(10_000)]
ci_boot = np.percentile(boot_means, [2.5, 97.5])
print(f"Bootstrap 95% CI: {ci_boot.round(3)}")

# Bootstrap CI for model accuracy (classification)
y_true = np.random.randint(0, 2, 1000)
y_pred = (np.random.rand(1000) > 0.4).astype(int)
boot_accs = []
for _ in range(5000):
    idx = rng.integers(0, 1000, 1000)
    boot_accs.append((y_true[idx] == y_pred[idx]).mean())
acc_ci = np.percentile(boot_accs, [2.5, 97.5])
print(f"Accuracy 95% CI: {acc_ci.round(4)}")
CLT Demo

Central Limit Theorem in Practice

import numpy as np
from scipy import stats

rng = np.random.default_rng(7)

# LLN: sample mean converges to true mean
for n in [10, 100, 1000, 10_000]:
    s = rng.exponential(scale=2, size=n)
    print(f"n={n:6d}  x̄={s.mean():.4f}  (true=2.0)")

# CLT: even for exponential data, means are normal
sample_means = [rng.exponential(2, 50).mean()
                for _ in range(10_000)]
_, p = stats.normaltest(sample_means)
print(f"\nCLT normality p={p:.4e}  (large → normal)")
print(f"Theoretical SE: {2/np.sqrt(50):.4f}")
print(f"Empirical   SE: {np.std(sample_means):.4f}")

# MLE as NLL minimization (equivalent to cross-entropy)
# P(y=1|x) = sigmoid(logit), loss = -log P(y|x)
def binary_nll(y_true, logits):
    """Negative log-likelihood = cross-entropy loss."""
    probs   = 1 / (1 + np.exp(-logits))
    probs   = np.clip(probs, 1e-10, 1-1e-10)
    return -np.mean(y_true*np.log(probs) + (1-y_true)*np.log(1-probs))

y = np.array([1,0,1,1,0], dtype=float)
logits = np.array([2.0,-1.5,1.0,0.5,-0.8])
print(f"Binary NLL (= BCE loss): {binary_nll(y, logits):.4f}")

Bayesian Inference

Principled uncertainty quantification — the mathematical framework behind Bayesian neural networks, RLHF priors, and probabilistic ML.

Theory

From Prior to Posterior — Principled Belief Update

Bayesian inference treats parameters θ as random variables, not fixed unknowns. We start with a prior p(θ) encoding beliefs before seeing data. After observing data D, we compute the posterior via Bayes' theorem: p(θ|D) ∝ p(D|θ)·p(θ). The term p(D|θ) is the likelihood. The normalizing constant p(D) = ∫p(D|θ)p(θ)dθ is the marginal likelihood or evidence.

Conjugate priors make this tractable analytically: if the prior and likelihood are in conjugate families, the posterior has the same form as the prior — just with updated hyperparameters. Beta–Binomial, Dirichlet–Categorical, and Normal–Normal are the most important conjugate pairs in ML.

MAP estimation (Maximum A Posteriori): θ̂_MAP = argmax_θ log p(D|θ) + log p(θ). The log-prior term acts as regularization: Gaussian prior → L2 (Ridge), Laplace prior → L1 (Lasso).

When posteriors are intractable, we use approximate inference: MCMC (exact in the limit, slow), Variational Inference (optimize a lower bound ELBO, fast), or Laplace approximation (Gaussian approximation at the MAP).

  • Posterior predictive: p(x*|D) = ∫p(x*|θ)p(θ|D)dθ — averages predictions over the full posterior, giving calibrated uncertainty.
  • ELBO = E_q[log p(x|z)] − KL(q(z)||p(z)) — the variational objective in VAEs, where q is the approximate posterior (encoder).
  • Bayesian model comparison: p(M|D) ∝ p(D|M)·p(M). The evidence p(D|M) naturally penalizes model complexity — Occam's razor is built in.

Conjugate Update

Beta-Binomial Model (Thompson Sampling)

Conjugate Update
Beta(α,β) + k successes, n−k failures → Beta(α+k, β+n−k) 
from scipy import stats
import numpy as np

# Prior: uniform over [0,1] — no knowledge
a0, b0 = 1, 1

# Observe 14 successes in 20 trials
k, n           = 14, 20
a_post, b_post = a0+k, b0+(n-k)
post           = stats.beta(a_post, b_post)

print(f"Posterior: Beta({a_post},{b_post})")
print(f"Mean    = {post.mean():.4f}")
print(f"Mode    = {(a_post-1)/(a_post+b_post-2):.4f}")
print(f"95% HDI = {post.ppf([0.025,0.975]).round(4)}")

# MAP = L2 regularized logistic regression equivalent
# MLE would give 14/20 = 0.7, MAP shrinks toward prior mean
mle = k/n
map_est = (k + a0 - 1) / (n + a0 + b0 - 2)
print(f"MLE={mle:.3f}  MAP={map_est:.3f}  Bayes={post.mean():.3f}")

Metropolis-Hastings + MAP vs MLE

import numpy as np
from scipy import stats

# Bayesian linear regression: L2 = MAP w/ Gaussian prior
np.random.seed(0)
n   = 50
X   = np.random.randn(n)
y   = 2*X + np.random.randn(n)

# Log-posterior for slope β (prior N(0,1), likelihood N(Xβ, 1))
def log_post(beta):
    log_lik  = stats.norm.logpdf(y, loc=X*beta, scale=1).sum()
    log_prior= stats.norm.logpdf(beta, 0, 1)   # Gaussian prior
    return log_lik + log_prior

# Metropolis-Hastings MCMC
rng     = np.random.default_rng(42)
beta    = 0.0
samples = []
for _ in range(20_000):
    prop  = beta + rng.normal(0, 0.15)
    log_a = log_post(prop) - log_post(beta)
    if np.log(rng.random()) < log_a:
        beta = prop
    samples.append(beta)

posterior = np.array(samples[5000:])  # burn-in
# Analytical MAP: (XᵀX + σ²/σ_prior²·I)⁻¹Xᵀy = Ridge!
map_exact = (X @ y) / (X @ X + 1)  # σ²=1, σ_prior²=1
print(f"MCMC posterior mean: {posterior.mean():.4f}")
print(f"MAP (Ridge) β̂:       {map_exact:.4f}")
print(f"MLE β̂:               {(X@y)/(X@X):.4f}")
print(f"95% HDI: {np.percentile(posterior,[2.5,97.5]).round(4)}")

Linear Algebra for ML

Matrices, eigenvalues, and decompositions are the computational engine of every neural network and dimensionality reduction algorithm.

Theory

Eigendecomposition, SVD, and Why They Dominate ML

A matrix A encodes a linear transformation. Its eigendecomposition A = VΛV⁻¹ reveals invariant directions (eigenvectors) and their scaling factors (eigenvalues λᵢ). For symmetric positive semi-definite matrices (covariance matrices), all eigenvalues are real and non-negative, eigenvectors are orthogonal.

SVD generalizes eigendecomposition to rectangular matrices: X = UΣVᵀ, where U ∈ ℝᵐˣᵐ contains left singular vectors (observation directions), Σ ∈ ℝᵐˣⁿ is diagonal (singular values σᵢ ≥ 0), and V ∈ ℝⁿˣⁿ contains right singular vectors (feature directions). The rank-k truncated SVD is the best rank-k approximation in both Frobenius and spectral norm.

PCA finds directions of maximum variance: it's equivalent to SVD of the centered data matrix, or eigendecomposition of the empirical covariance. The principal components are orthogonal, and the first k explain the most variance.

The pseudo-inverse X⁺ = VΣ⁺Uᵀ is the least-squares solution to Xβ=y when X is singular or rectangular. The Cholesky decomposition Σ=LLᵀ provides a numerically stable way to sample from multivariate normals and solve positive-definite linear systems.

  • Spectral norm ||W||₂ = σ_max(W) — used in spectral normalization of GAN discriminators to enforce Lipschitz continuity.
  • Nuclear norm = Σσᵢ — convex relaxation of matrix rank; used in matrix completion (Netflix prize) and low-rank regularization.
  • Matrix calculus: ∂/∂W(||Xw−y||²) = 2Xᵀ(Xw−y). Jacobians and Hessians underlie backpropagation in every neural network.

Information Theory

Shannon's framework for quantifying information — the mathematical foundation of LLM loss functions, compression, and representation learning.

Theory

Entropy, Divergence, and the Limits of Compression

Shannon entropy H(X) = −Σ p(x)·log₂p(x) measures the average surprise or uncertainty in a random variable X. It equals the minimum expected number of bits needed to encode samples from X (source coding theorem). Maximum entropy (log₂K bits for K outcomes) is achieved by the uniform distribution.

Cross-entropy H(p,q) = −Σ p(x)·log q(x) measures the expected code length when using a model distribution q to encode data from the true distribution p. It decomposes as H(p,q) = H(p) + D_KL(p||q). Since H(p) is constant with respect to q, minimizing cross-entropy IS minimizing KL divergence from true to model distribution — this is exactly what LLM training does.

KL Divergence D_KL(P||Q) = Σ P(x)·log(P(x)/Q(x)) ≥ 0 (Gibbs' inequality) measures how much information is lost when using Q to approximate P. Asymmetric: D_KL(P||Q) ≠ D_KL(Q||P). Forward KL (P||Q) is mean-seeking; reverse KL (Q||P) is mode-seeking.

Mutual Information I(X;Y) = H(X) − H(X|Y) = H(Y) − H(Y|X) = D_KL(P(X,Y)||P(X)P(Y)) measures how much knowing Y reduces uncertainty about X. I(X;Y) = 0 iff X and Y are independent.

  • Perplexity = 2^H(p,q) in bits (or e^H in nats) — the standard LLM evaluation metric. A perplexity of K means the model is "as confused as if choosing uniformly among K options."
  • Jensen-Shannon Divergence JSD(P||Q) = ½D_KL(P||M) + ½D_KL(Q||M), M=(P+Q)/2 — symmetric, bounded in [0,1], used in GAN training (original GAN loss = JSD).
  • Bits-back coding / minimum description length: connects Bayesian model selection to compression — a model is good if it compresses the data well.

Regression & GLMs

Linear regression and its extensions — the interpretable workhorse of ML and the probabilistic basis for all generalized linear models.

Theory

OLS, Gauss-Markov, and the Bias-Variance Tradeoff

Ordinary Least Squares (OLS) finds β̂ that minimizes the residual sum of squares ||y − Xβ||². The normal equations (XᵀX)β = Xᵀy have the closed-form solution β̂ = (XᵀX)⁻¹Xᵀy. The Gauss-Markov theorem guarantees β̂_OLS is the Best Linear Unbiased Estimator (BLUE) when errors are i.i.d. with E[ε]=0 and Var(ε)=σ²I.

Statistical properties: β̂ ~ Normal(β, σ²(XᵀX)⁻¹) under normal errors. R² = 1 − SS_res/SS_tot measures fraction of variance explained. Adjusted R² = 1 − (1−R²)(n−1)/(n−p−1) penalizes for extra predictors.

Regularization and MAP: Ridge regression (L2) = MAP with Gaussian prior N(0,1/λ·I). Lasso (L1) = MAP with Laplace prior. Ridge shrinks all coefficients uniformly; Lasso produces sparse solutions by setting irrelevant coefficients to exactly zero. ElasticNet combines both.

Generalized Linear Models (GLMs) extend OLS to non-Gaussian responses: logistic regression (Binomial/Bernoulli), Poisson regression (count data), Gamma regression (positive continuous). The link function maps the linear predictor to the response mean: logit for binary, log for counts.

  • Multicollinearity: when features are correlated, (XᵀX) is near-singular. Ridge regression adds λI to stabilize: (XᵀX+λI)⁻¹. Condition number κ = σ_max/σ_min measures how close to singular.
  • Heteroscedasticity: non-constant error variance violates Gauss-Markov. Weighted least squares or robust regression (Huber loss) are corrections used in financial analytics.
  • The hat matrix H = X(XᵀX)⁻¹Xᵀ projects y onto the column space of X. Diagonal elements hᵢᵢ = leverage — points with high leverage can unduly influence the fit.

import numpy as np
from scipy import stats

np.random.seed(0)
n = 200
X_raw = np.random.randn(n, 3)
beta_true = np.array([2.0, -1.5, 0.8])
y = X_raw @ beta_true + np.random.randn(n)*0.5
X = np.column_stack([np.ones(n), X_raw])   # intercept

# Closed-form OLS
beta_hat = np.linalg.solve(X.T @ X, X.T @ y)
y_hat    = X @ beta_hat
resid    = y - y_hat
ss_res   = resid @ resid
ss_tot   = ((y - y.mean())**2).sum()
R2       = 1 - ss_res/ss_tot
R2_adj   = 1 - (1-R2)*(n-1)/(n-X.shape[1]-1)
print(f"β̂ = {beta_hat.round(3)}")
print(f"R²={R2:.4f}  R²_adj={R2_adj:.4f}")

# Standard errors + t-tests + p-values
sigma2    = ss_res / (n - X.shape[1])
var_beta  = sigma2 * np.linalg.inv(X.T @ X)
se        = np.sqrt(np.diag(var_beta))
t_stat    = beta_hat / se
p_val     = 2 * stats.t.sf(np.abs(t_stat), df=n-len(beta_hat))
for i,(b,s,t,p) in enumerate(zip(beta_hat,se,t_stat,p_val)):
    sig = "***" if p<0.001 else "**" if p<0.01 else "*" if p<0.05 else ""
    print(f"β{i}={b:7.3f}  SE={s:.3f}  t={t:6.2f}  p={p:.4f} {sig}")

Hypothesis Testing

Rigorous statistical decision-making for A/B tests, model comparisons, feature selection, and production monitoring.

Theory

Null Hypotheses, p-values, Power, and Multiple Testing

A hypothesis test starts with a null hypothesis H₀ (e.g., "model A and B have equal accuracy") and an alternative hypothesis H₁. We compute a test statistic T from data, and the p-value = P(|T| ≥ t_obs | H₀) — the probability of observing data this extreme if H₀ were true. We reject H₀ when p < α (significance level, typically 0.05).

Type I error (false positive): rejecting H₀ when it's true. Rate = α. Type II error (false negative): failing to reject H₀ when H₁ is true. Rate = β. Power = 1−β = P(reject H₀ | H₁ is true). A well-powered study (≥ 0.8) has enough samples to detect the effect of interest.

Effect size (e.g., Cohen's d) quantifies the magnitude of an effect independent of sample size. A statistically significant result with a tiny effect size may not be practically meaningful. Always report both p-values AND effect sizes.

Multiple comparisons: testing K hypotheses inflates the family-wise error rate (FWER). Bonferroni correction uses α/K per test (conservative). Benjamini-Hochberg FDR controls the expected fraction of false discoveries among rejections (less conservative, preferred in ML research).

  • t-tests: compare means. One-sample (vs. fixed value), two-sample independent (A/B test), paired (before/after). Welch's t-test handles unequal variances.
  • Chi-squared test: tests independence in contingency tables. Used for categorical feature vs. categorical label associations.
  • ANOVA (F-test): compares means across ≥3 groups. One-way (one factor), two-way (two factors). Post-hoc tests (Tukey HSD) identify which pairs differ.
  • Non-parametric tests: Mann-Whitney U (two-sample, no normality assumption), Kruskal-Wallis (non-parametric ANOVA), Wilcoxon signed-rank (paired, robust).

from scipy import stats
from statsmodels.stats.multitest import multipletests
import numpy as np

rng = np.random.default_rng(42)

# ── 1. A/B test: Welch t-test (unequal variance) ──
ctrl = rng.normal(10, 2, 1000)     # control group
exp  = rng.normal(10.3, 2.2, 800)  # smaller treatment group
t, p = stats.ttest_ind(ctrl, exp, equal_var=False)
d    = (exp.mean()-ctrl.mean()) / np.sqrt((ctrl.var(ddof=1)+exp.var(ddof=1))/2)
print(f"A/B test: t={t:.3f}  p={p:.4f}  Cohen's d={d:.4f}")

# ── 2. McNemar test: compare two classifiers ──
# n01 = model A wrong, B right; n10 = A right, B wrong
n00, n01, n10, n11 = 800, 40, 25, 135   # 1000 test samples
chi2_mcnemar = (abs(n01-n10)-1)**2 / (n01+n10)  # continuity corrected
p_mcnemar    = 1 - stats.chi2.cdf(chi2_mcnemar, df=1)
print(f"McNemar: χ²={chi2_mcnemar:.3f}  p={p_mcnemar:.4f}")
print(f"Model A acc={(n00+n10)/1000:.3f}  Model B acc={(n00+n01)/1000:.3f}")

# ── 3. KS test for data drift detection ──
train_dist = rng.normal(0, 1, 5000)    # training distribution
prod_nodrift = rng.normal(0, 1, 500)  # no drift
prod_drift   = rng.normal(0.5, 1.3, 500)  # drifted!
ks_ok,  p_ok  = stats.ks_2samp(train_dist, prod_nodrift)
ks_bad, p_bad = stats.ks_2samp(train_dist, prod_drift)
print(f"KS no-drift: D={ks_ok:.4f}  p={p_ok:.4f}  alert={'YES' if p_ok<0.05 else 'NO'}")
print(f"KS drifted:  D={ks_bad:.4f}  p={p_bad:.4e}  alert={'YES' if p_bad<0.05 else 'NO'}")

# ── 4. Multiple testing: feature selection with BH correction ──
n_features = 500
X = rng.normal(0, 1, (200, n_features))
# Only first 10 features are truly related to y
y = X[:,:10] @ rng.normal(0, 1, 10) + rng.normal(0, 1, 200)

p_values = np.array([
    stats.pearsonr(X[:,j], y).pvalue for j in range(n_features)
])
# Bonferroni: very conservative
reject_bonf = p_values < 0.05/n_features
# BH FDR: less conservative, more power
reject_bh, _, _, _ = multipletests(p_values, method='fdr_bh', alpha=0.05)
print(f"\nBonferroni selected: {reject_bonf.sum()} features")
print(f"BH FDR selected:     {reject_bh.sum()} features")
print(f"True features found (BH): {reject_bh[:10].sum()} / 10")

# Compare two models' accuracy with 95% CI on the difference
model_a_correct = rng.binomial(1, 0.82, 1000)
model_b_correct = rng.binomial(1, 0.85, 1000)
diff_obs = model_b_correct.mean() - model_a_correct.mean()

boot_diffs = []
for _ in range(10_000):
    idx = rng.integers(0, 1000, 1000)
    boot_diffs.append(
        model_b_correct[idx].mean() - model_a_correct[idx].mean()
    )

Sampling Methods

Monte Carlo, MCMC, and importance sampling — the computational engines behind Bayesian inference, generative models, and LLM decoding.

Theory

Monte Carlo, Importance Sampling, and Stochastic Approximation

Monte Carlo estimation approximates integrals and expectations by sampling: E_p[f(X)] ≈ (1/N)·Σf(xᵢ), xᵢ~p. The error is O(1/√N) regardless of dimension — Monte Carlo breaks the curse of dimensionality for integration. The estimate is unbiased, and variance decreases as N increases (Law of Large Numbers).

Importance Sampling estimates E_p[f(X)] using samples from a different distribution q: E_p[f(X)] = E_q[f(X)·w(X)], where w(X)=p(X)/q(X) is the importance weight. Critical for estimating rare-event probabilities, estimating expectations under a policy different from the data collection policy (off-policy RL), and KL divergence estimation.

MCMC (Markov Chain Monte Carlo): when we can't sample from p(x) directly, we construct a Markov chain with stationary distribution p. Metropolis-Hastings: propose x' from q(x'|x), accept with probability min(1, p(x')q(x|x')/(p(x)q(x'|x))). Gibbs sampling: cycle through coordinates, sampling each from its conditional. Hamiltonian Monte Carlo (NUTS in PyMC/Stan): use gradient information for efficient proposals in high dimensions.

  • Rejection sampling: sample from q, accept with probability p(x)/(M·q(x)). Inefficient in high dimensions — used in discrete diffusion sampling.
  • Ancestral sampling in autoregressive models: sample x₁~p(x₁), then x₂~p(x₂|x₁), etc. This is exactly LLM text generation with greedy/temperature/top-k/top-p sampling strategies.
  • Stratified and quasi-Monte Carlo: use structured low-discrepancy sequences (Halton, Sobol) instead of random samples — faster convergence for high-dimensional integration.

import numpy as np
from scipy import stats

rng = np.random.default_rng(0)

# ── MC Integration ──
N = 1_000_000
x,y = rng.uniform(-1,1,N), rng.uniform(-1,1,N)
pi_est = 4*(x**2+y**2<1).mean()
print(f"π ≈ {pi_est:.6f}")

# ── LLM decoding strategies ──
vocab_size = 10
logits = np.random.randn(vocab_size)

def top_k_sampling(logits, k=5, T=1.0):
    scaled = logits / T
    top_k_idx = np.argsort(scaled)[-k:]
    probs = np.exp(scaled[top_k_idx]); probs /= probs.sum()
    return np.random.choice(top_k_idx, p=probs)

def top_p_sampling(logits, p=0.9, T=1.0):
    scaled = logits / T
    probs  = np.exp(scaled - scaled.max()); probs /= probs.sum()
    sorted_idx   = np.argsort(probs)[::-1]
    cumulative   = np.cumsum(probs[sorted_idx])
    cutoff       = np.searchsorted(cumulative, p) + 1
    nucleus_idx  = sorted_idx[:cutoff]
    nucleus_probs= probs[nucleus_idx]; nucleus_probs /= nucleus_probs.sum()
    return np.random.choice(nucleus_idx, p=nucleus_probs)

tk = top_k_sampling(logits, k=3)
tp = top_p_sampling(logits, p=0.9)
print(f"Top-k sample: token {tk}  Top-p sample: token {tp}")

# ── Importance sampling (PPO ratio) ──
log_pi_new = np.array([-1.0, -2.0, -0.5])  # log policy probs
log_pi_old = np.array([-1.2, -1.8, -0.7])
ratio      = np.exp(log_pi_new - log_pi_old)  # IS ratio r(θ)
print(f"PPO IS ratios: {ratio.round(4)}")
eps = 0.2
ratio_clipped = np.clip(ratio, 1-eps, 1+eps)  # PPO clipping
print(f"Clipped ratios: {ratio_clipped.round(4)}")

# ── Reparameterization trick (VAE / diffusion) ──
def reparameterize(mu, log_var):
    """Sample z ~ N(mu, exp(log_var)) via reparameterization.
    Enables backprop through the sampling operation."""
    std = np.exp(0.5 * log_var)     # σ = exp(log σ²/2)
    eps = np.random.randn(*mu.shape)  # ε ~ N(0,I) — constant wrt θ
    return mu + std * eps              # z = μ + σ·ε (differentiable!)

# Simulate VAE encoder output
batch_size, latent_dim = 32, 128
mu      = np.random.randn(batch_size, latent_dim)     # encoder mean
log_var = np.random.randn(batch_size, latent_dim) - 2 # encoder log-var
z       = reparameterize(mu, log_var)

# KL divergence to N(0,I) prior (closed-form for Gaussian)
kl_loss = -0.5 * np.mean(
    np.sum(1 + log_var - mu**2 - np.exp(log_var), axis=-1)
)
print(f"\nVAE: z shape={z.shape}  KL loss={kl_loss:.4f}")
print(f"     z mean≈{z.mean():.4f}  z std≈{z.std():.4f}")

# DDPM noise schedule (forward diffusion)
T_steps = 1000
betas   = np.linspace(1e-4, 0.02, T_steps)    # linear schedule
alphas  = 1 - betas
alpha_bar = np.cumprod(alphas)                  # cumulative product
# At step t: x_t = sqrt(ᾱ_t)·x_0 + sqrt(1-ᾱ_t)·ε
x0  = np.random.randn(64, 3, 32, 32)    # clean image
t   = 500
eps = np.random.randn(*x0.shape)         # noise
x_t = np.sqrt(alpha_bar[t])*x0 + np.sqrt(1-alpha_bar[t])*eps
print(f"\nDDPM t={t}: ᾱ={alpha_bar[t]:.4f}  SNR={alpha_bar[t]/(1-alpha_bar[t]):.4f}")
